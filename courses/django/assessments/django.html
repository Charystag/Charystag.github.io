Django Exercise: Mini Blog Application

Objective

This exercise will help you develop a Django application using the Model-View-Template (MVT) architecture. You will create a functional mini-blog application that includes multiple models, views, templates, and URL configurations. You will also implement redirections between views and templates and ensure good development practices such as excluding migrations and database files.

Scenario

You are tasked with building a Mini Blog Application that allows users to:

  • Create, edit, view, and delete blog posts.
  • Add, edit, and delete comments on blog posts.
  • Redirect between views and use templates that extend a common base layout.

Your application must have test coverage to ensure it functions as expected.

Specifications

Step 1: Models

You will create two models:

  1. BlogPost:

    • Fields:
      • title: The title of the blog post (CharField, max length: 100).
      • content: The content of the blog post (TextField).
      • created_at: Timestamp for when the blog post was created (DateTimeField, auto_now_add=True).
    • Features:
      • A __str__ method that returns the post title.

  2. Comment:

    • Fields:
      • post: ForeignKey to BlogPost (on_delete=models.CASCADE).
      • author: The name of the comment author (CharField, max length: 50).
      • text: The content of the comment (TextField).
      • created_at: Timestamp for when the comment was created (DateTimeField, auto_now_add=True).
    • Features:
      • A __str__ method that returns a truncated version of the comment text.

Step 2: Views

You will create the following views:

  1. List Blog Posts:

    • Displays all blog posts.
    • Template: post_list.html.
    • Features:
      • Each blog post has "View," "Edit," and "Delete" links.
      • Include a "Create New Post" button that redirects to the create post view.

  2. View Blog Post Details:

    • Displays a blog post and its associated comments.
    • Template: post_detail.html.
    • Features:
      • Each comment has "Edit" and "Delete" links.
      • Include a link to add a comment to the blog post.
      • Include a "Back to Blog List" button.

  3. Create Blog Post:

    • Displays a form for creating a new blog post.
    • Template: create_post.html.
    • Features:
      • Redirect to the blog post list after successfully creating a post.
      • Include a "Cancel" button that redirects to the blog post list.

  4. Edit Blog Post:

    • Displays a form for editing an existing blog post.
    • Template: edit_post.html.
    • Features:
      • Pre-fill the form with the existing blog post's data.
      • Redirect to the blog post detail view after saving changes.
      • Include a "Cancel" button.

  5. Add Comment to Blog Post:

    • Displays a form for adding a comment to a blog post.
    • Template: add_comment.html.
    • Features:
      • Redirect to the blog post detail view after adding a comment.
      • Include a "Cancel" button.

  6. Edit Comment:

    • Displays a form for editing an existing comment.
    • Template: edit_comment.html.
    • Features:
      • Pre-fill the form with the existing comment's data.
      • Redirect to the blog post detail view after saving changes.
      • Include a "Cancel" button.

  7. Delete Blog Post:

    • Deletes a specific blog post.
    • Redirects to the blog post list after deletion.
    • Includes a confirmation prompt.

  8. Delete Comment:

    • Deletes a specific comment.
    • Redirects to the blog post detail view after deletion.
    • Includes a confirmation prompt.

Step 3: Templates

Templates should extend a common base.html template. Include a navigation bar with the following links:

  • Home: Redirects to the blog post list.
  • Create Post: Redirects to the create post view.

Each page should have a consistent layout and include clear navigation. The templates should support the following outputs:

Example Features

  1. Post List (/):

    • Blog Posts: 
      1. Django Basics (View | Edit | Delete)
2. Advanced Django Features (View | Edit | Delete)
    • [Create New Post]

  2. Post Detail (/post/<id>/):

    • Title: Django Basics
    • Content: Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design.
    • Comments: 
      - Author: Alice, Text: Great introduction to Django! (Edit | Delete)
- Author: Bob, Text: Very helpful! (Edit | Delete)
    • [Add Comment]
    • [Back to Blog List]

  3. Add Post (/post/create/):

    • Title: [Input Field]
    • Content: [Text Area]
    • [Save Post] [Cancel]

  4. Add Comment (/post/<id>/add_comment/):

    • Author: [Input Field]
    • Text: [Text Area]
    • [Add Comment] [Cancel]

  5. Edit Post (/post/<id>/edit/):

    • Title: Django Basics (Pre-filled in input field)
    • Content: Django is a high-level Python web framework... (Pre-filled in text area)
    • [Save Changes] [Cancel]

  6. Edit Comment (/comment/<id>/edit/):

    • Text: Great introduction to Django! (Pre-filled in input field)
    • [Save Changes] [Cancel]

  7. Delete Post Confirmation (/post/<id>/delete/):

    • Are you sure you want to delete "Django Basics"?
    • [Yes, Delete] [Cancel]

  8. Delete Comment Confirmation (/comment/<id>/delete/):

    • Are you sure you want to delete the comment by Alice: "Great introduction to Django!"?
    • [Yes, Delete] [Cancel]

Step 4: URLs

Configure the following URLs:

  • /: Blog post list (List view).
  • /post/<id>/: Blog post detail (Detail view).
  • /post/create/: Create a new blog post (Create view).
  • /post/<id>/edit/: Edit an existing blog post (Edit view).
  • /post/<id>/add_comment/: Add a comment to a blog post (Add Comment view).
  • /comment/<id>/edit/: Edit an existing comment (Edit Comment view).
  • /post/<id>/delete/: Delete a blog post (Delete Post view).
  • /comment/<id>/delete/: Delete a comment (Delete Comment view).

Step 5: Tests

Write 10 test cases to ensure:

  1. Blog posts are created, edited, and deleted correctly.
  2. Comments are added, edited, and deleted correctly.
  3. Views return expected HTTP responses.
  4. Templates render the correct data.
  5. Redirections occur as intended.

      • All project files (excluding migrations and database files).
      • A README.md file with:
        • A description of the project.
        • Instructions to set up and run the project locally.
      • A requirements.txt file specifying dependencies.
