SQL Exercise - University Enrollment Management System

Objective:
Design and implement a relational database to manage students, courses, and enrollments in a university. The system will support features like tracking enrollments, course capacities, and student-specific statistics. Queries will ensure robust testing of relational database concepts and SQL skills.

Challenge:

You must complete the following tasks to build and query the database. Ensure that you use the exact table and column names specified to maintain compatibility with testing.

  1. Task 1: Design the Database Schema
    • Create the following tables:
      • Students: Tracks student details.
        • Columns: student_id (INT PRIMARY KEY), name (VARCHAR(50)), age (INT), gender (VARCHAR(10)).
      • Courses: Stores information about university courses.
        • Columns: course_id (INT PRIMARY KEY), course_name (VARCHAR(50)), credits (INT), capacity (INT).
      • Enrollments: Links students to courses.
        • Columns: enrollment_id (INT PRIMARY KEY AUTO_INCREMENT), student_id (INT FOREIGN KEY REFERENCES Students(student_id)), course_id (INT FOREIGN KEY REFERENCES Courses(course_id)).
  1. Task 2: Populate Tables with Sample Data
    • Insert data into the tables to satisfy these conditions:
      • At least 5 students are registered.
      • At least 4 courses are available, each with a specified capacity.
      • At least 8 enrollments exist, including some courses reaching their capacity.
  1. Task 3: Query Student Enrollment Details
    • Write SQL queries to perform the following:
      • Query 1: Retrieve the list of all students and the courses they are enrolled in, including the course name and credits.
      • Query 2: Find students who are not enrolled in any course.
  1. Task 4: Query Course Statistics
    • Write SQL queries to extract the following information:
      • Query 3: List all courses and the number of students enrolled in each course.
      • Query 4: Identify courses where the number of enrollments exceeds half the course capacity.
  1. Task 5: Advanced Enrollment Analysis
    • Write SQL queries to answer these questions:
      • Query 5: Find students enrolled in the maximum number of courses.
      • Query 6: Calculate the total credits each student is currently taking.
      • Query 7: Identify courses with no enrollments.
  1. Task 6: Implement Constraints
    • Add the following constraints to your schema:
      • Prevent enrolling students in a course if it has reached its capacity.
      • Ensure each student can enroll in a maximum of 5 courses.
  1. Task 7: Clean and Reset Data
    • Write SQL scripts to:
      • Remove all enrollments for a specific course without deleting the course itself.
      • Delete all students who have not enrolled in any courses.

Mandatory Table and Column Names for Testing:

  • Students: student_id, name, age, gender.
  • Courses: course_id, course_name, credits, capacity.
  • Enrollments: enrollment_id, student_id, course_id.

Questions for Reflection:

  • How do foreign keys enforce relationships between tables, and how did they impact your design?
  • What considerations must be made when implementing constraints like course capacity?
  • How could you optimize queries for large datasets with thousands of students and enrollments?

Skills Practiced:

  • Relational database design (schema creation).
  • Writing queries for data retrieval and analysis.
  • Implementing constraints for data integrity.
  • Understanding and applying JOIN operations.

  1. Create a GitHub repository named LibraryManagementSystem.

  3. Push all files to the repository.
  4. Submit the GitHub repository link.
