Running SQL Commands with Docker: A Step-by-Step Guide

This document explains how to use Docker to run SQL commands without the need to install additional software beyond Docker. We will use MySQL as the database engine, leveraging Docker containers to host the MySQL server and client.

Prerequisites

  • Docker Installed: Ensure Docker is installed and running on your system.
  • Basic Docker Knowledge: Familiarity with basic Docker commands is helpful but not mandatory.

Steps to Run SQL Commands Using Docker

1. Set Up a Docker Network

To enable communication between the MySQL server and client containers, create a Docker network:

docker network create net

2. Run a MySQL Server Container

Run the MySQL server container with the following command:

docker run -e MYSQL_ROOT_PASSWORD=bonjour -p 3306:3306 -d --network net --network-alias serv --name server mysql:latest

Explanation of Flags:

  • -e MYSQL_ROOT_PASSWORD=bonjour: Sets the root password for MySQL as bonjour.
  • -p 3306:3306: Maps port 3306 of the container to port 3306 on your host machine (optional if not accessing MySQL externally).
  • -d: Runs the container in detached mode.
  • --network net: Connects the container to the network created earlier.
  • --network-alias serv: Assigns the alias serv to this container within the Docker network.
  • --name server: Names the container server.
  • mysql:latest: Specifies the MySQL image to use (latest version).

3. Connect to the MySQL Server Using a MySQL Client Container

To connect to the MySQL server and execute SQL commands, run the following command:

docker run -it --network net --network-alias client mysql:latest mysql -h serv.net -u root -pbonjour

Explanation of Flags:

  • -it: Runs the container interactively with a terminal.
  • --network net: Connects the container to the same network as the server.
  • --network-alias client: Assigns the alias client to this container within the network.
  • mysql:latest: Specifies the MySQL image to use (latest version).
  • mysql -h serv.net -u root -pbonjour: Launches the MySQL client and connects to the server (serv.net) as the root user with the password bonjour.

4. Run SQL Commands in the Terminal

Once the client container is running, you will be inside the MySQL command-line interface. You can now execute SQL commands directly. For example:

SHOW DATABASES;
CREATE DATABASE test_db;
USE test_db;
CREATE TABLE users (id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(255));
INSERT INTO users (name) VALUES ('Alice'), ('Bob');
SELECT * FROM users;

Advantages of This Setup

  1. No Local Software Installation: You don’t need to install MySQL on your host machine—everything runs inside Docker containers.
  2. Isolation: The MySQL server and client run in isolated environments, preventing interference with your system.
  3. Ease of Cleanup: Remove all containers and networks with a single command when done.

Cleanup

To stop and remove the containers and network after you're finished, use the following commands:

docker rm -f server
docker rm -f $(docker ps -a -q --filter "name=client")
docker network rm net

Conclusion

By leveraging Docker, you can set up and interact with a MySQL database in minutes without installing any database software locally. This approach is excellent for development, testing, and learning SQL in a clean and portable environment.

    python3 --version

  • pip Installed: Verify that pip is installed:

    pip --version

A virtual environment isolates project-specific dependencies. Use the venv module to create one:

  1. Navigate to your project directory or create a new one:

    mkdir django_project
cd django_project

  2. Create a virtual environment:

    python3 -m venv venv

  3. Activate the virtual environment:

    • On Linux/Mac: 
      . venv/bin/activate
    • On Windows: 
      venv\Scripts\activate

    Once activated, your terminal prompt will change to indicate the virtual environment is active, typically showing (venv).

2. Install Django in the Virtual Environment

With the virtual environment activated, install Django using pip:

pip install django

To confirm Django is installed, run:

django-admin --version

This command should output the version of Django you installed.

3. Test Your Django Installation

  1. Create a Django Project:

    Run the following command to create a new Django project:

from .models import Question, Choice

def vote(request, question_id):
    question = get_object_or_404(Question, pk=question_id)
    try:
        selected_choice = question.choice_set.get(pk=request.POST['choice'])
    except (KeyError, Choice.DoesNotExist):

        selected_choice.save()
