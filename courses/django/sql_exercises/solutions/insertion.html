Django Book

Solution

  1. Insert Data: Add the three specified records to the books table. Use a single query with multiple VALUES blocks for efficiency:

    INSERT INTO books (title, author, published_year, genre) 
VALUES 
    ('The Great Gatsby', 'F. Scott Fitzgerald', 1925, 'Fiction'),
    ('1984', 'George Orwell', 1949, 'Dystopian'),
    ('To Kill a Mockingbird', 'Harper Lee', 1960, 'Fiction');

    Explanation:

    • The INSERT INTO statement specifies the table name (books) and the columns to insert values into (title, author, published_year, genre).
    • Each row of data is enclosed in parentheses, separated by commas.

  2. Verify the Data: Use the following query to confirm the data has been inserted correctly:

    SELECT * FROM books;

    Expected Output:

    idtitleauthorpublished_yeargenre
    1The Great GatsbyF. Scott Fitzgerald1925Fiction
    21984George Orwell1949Dystopian
    3To Kill a MockingbirdHarper Lee1960Fiction

    Note: The id column will auto-increment, so its values will depend on the current state of the table.

  3. Optional Extension: Add more records of your choice to the table:

    INSERT INTO books (title, author, published_year, genre) 
VALUES 
    ('Pride and Prejudice', 'Jane Austen', 1813, 'Romance'),
    ('Moby Dick', 'Herman Melville', 1851, 'Adventure');

    Query the table again to view all records:

    SELECT * FROM books;

Reflection Questions

  1. What SQL syntax is used to insert multiple rows into a table in one query?

    • The INSERT INTO ... VALUES statement allows multiple rows to be added by including each row's values in separate parentheses, separated by commas.

  2. How does MySQL handle data types when inserting records (e.g., text vs integers)?

    • MySQL automatically converts values to the appropriate data types, as long as they are compatible. For example:
      • Strings are stored as VARCHAR (or other text types) and enclosed in single quotes.
      • Integers are stored as INT and do not require quotes.
      • Errors occur if a value is incompatible (e.g., inserting text into an integer column).

  3. What does SELECT * FROM books; return, and why is it useful?

    • The SELECT * query retrieves all columns and rows from the books table.
    • It is useful for verifying the content of the table after insertion or updates.
