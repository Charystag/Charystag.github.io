1. Create the Tables

products Table:

CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    category VARCHAR(255)
);

orders Table:

CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_date DATE NOT NULL,
    customer_name VARCHAR(255) NOT NULL
);

order_details Table:

CREATE TABLE order_details (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL,
    FOREIGN KEY (order_id) REFERENCES orders(id),
    FOREIGN KEY (product_id) REFERENCES products(id)
);

3. Write Queries

Calculate Total Revenue:

SELECT SUM(products.price * order_details.quantity) AS total_revenue
FROM order_details
INNER JOIN products ON order_details.product_id = products.id;

Explanation:

  • Joins order_details with products to access the price of each product.
  • Calculates the total cost for each order detail (price * quantity).
  • Sums up the total costs using SUM.

List Orders with Total Cost:

SELECT orders.customer_name, orders.order_date, 
       SUM(products.price * order_details.quantity) AS total_cost
FROM orders
INNER JOIN order_details ON orders.id = order_details.order_id
INNER JOIN products ON order_details.product_id = products.id
GROUP BY orders.id;

Explanation:

  • Joins orders, order_details, and products to access customer names, order dates, and product prices.
  • Groups by orders.id to calculate the total cost for each order.

Create an Index on category and Explain Its Benefit:

CREATE INDEX idx_category ON products(category);

Benefit:

  • The index on category speeds up queries filtering by category because it organizes data in a way that allows MySQL to locate rows more efficiently without scanning the entire table.

Example Query:

SELECT * FROM products WHERE category = 'Electronics';

Without an index, MySQL performs a full table scan. With an index, it directly accesses the relevant rows using the indexed category.

Optional Extensions

Identify the Most Popular Product:

SELECT products.name, SUM(order_details.quantity) AS total_quantity_sold
FROM products
INNER JOIN order_details ON products.id = order_details.product_id
GROUP BY products.id
ORDER BY total_quantity_sold DESC
LIMIT 1;

Explanation:

  • Groups order_details by product_id to calculate the total quantity sold for each product.
  • Orders by total quantity sold in descending order and limits the result to the most popular product.

Retrieve Total Revenue for a Specific Category:

SELECT products.category, SUM(products.price * order_details.quantity) AS category_revenue
FROM products
INNER JOIN order_details ON products.id = order_details.product_id
WHERE products.category = 'Electronics'
GROUP BY products.category;

Explanation:

  • Filters rows by the specified category.
  • Calculates total revenue for the category using SUM.

Reflection Questions

  1. How do foreign keys ensure data integrity in this schema?

    • Foreign keys enforce valid relationships between order_details, orders, and products, preventing orphaned records (e.g., an order detail referencing a non-existent product).

  2. Why are indexes particularly useful for columns like category?

    • Columns frequently used in filtering or grouping (like category) benefit from indexes, which allow faster lookups and reduced query execution time.

  3. How do aggregate functions like SUM enhance the ability to extract business insights?

    • Aggregate functions summarize data, enabling insights such as total revenue, average spending, or product popularity, which are crucial for decision-making.
