Joining Tables in MySQL

Objective:

Practice combining data from multiple tables using SQL JOIN statements.

Challenge:

  1. Create the authors Table:

    • Create a new table named authors with the following columns:
      • id: Primary Key, Auto-Increment, Integer.
      • name: Text.
      • birth_year: Integer.

  2. Insert Author Data:

    • Insert the following records into the authors table:
      namebirth_year
      F. Scott Fitzgerald1896
      George Orwell1903
      Harper Lee1926

  3. Perform an INNER JOIN:

    • Write a query to retrieve all books along with the corresponding authors’ birth years by joining the books table and the authors table.

  4. Optional Extension:

    • Add a new column author_id to the books table to reference the id from the authors table as a foreign key.
    • Update the books table to establish the relationship and re-run the JOIN query.

Questions for Reflection:

  • What is the purpose of an INNER JOIN in combining tables?
  • How does a foreign key enhance the relationship between tables?
  • How can data redundancy be minimized in relational databases?

Skills Practiced:

  • Creating relationships between tables.
  • Using JOIN to retrieve related data.
  • Managing foreign keys for data integrity.

Documentation Links

  • MySQL JOIN documentation
  • MySQL Foreign Key documentation
Hints
  • To create the authors table, use: 
    CREATE TABLE authors (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255),
    birth_year INT
);
  • To insert data into authors, use: 
    INSERT INTO authors (name, birth_year) 
VALUES ('F. Scott Fitzgerald', 1896), ('George Orwell', 1903), ('Harper Lee', 1926);
  • To perform an INNER JOIN to combine books with authors’ birth years, use: 
    SELECT books.title, authors.name, authors.birth_year 
FROM books 
INNER JOIN authors ON books.author = authors.name;
  • For the optional extension, add a column to books: 
    ALTER TABLE books ADD COLUMN author_id INT;
    Then update the author_id values and modify the JOIN query accordingly.
