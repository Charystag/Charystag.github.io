Django REST Framework Tutorial: Part 4 - Authentication and Permissions

This tutorial introduces authentication and permissions in Django REST Framework (DRF). These features control access to API endpoints, ensuring only authorized users can perform certain actions.

This guide is based on the official Django REST Framework Tutorial: Part 4 - Authentication and Permissions, with additional explanations to clarify the concepts.

1. What Are Authentication and Permissions?

Authentication:

  • Identifies the user making a request.
  • Common methods include:
    • Token-based authentication
    • Session authentication
    • Third-party methods like OAuth2

Permissions:

  • Define what authenticated users are allowed to do.
  • Examples:
    • Allow only the owner of a resource to edit or delete it.
    • Restrict certain views to admin users.

2. Adding Authentication to the Project

DRF provides several built-in authentication schemes. For this tutorial, the default session-based authentication will be used.

Update settings.py

Ensure rest_framework is configured with default authentication and permission classes:

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.SessionAuthentication',
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ],
}

Key Points:

  1. SessionAuthentication:
    • Relies on Django's session framework.
    • Suitable for browser-based interactions.
  2. IsAuthenticated:
    • Restricts access to authenticated users only.

3. Testing Authentication

Enable Admin and Login

  • Create a superuser to access the admin interface:

    python manage.py createsuperuser

  • Add the login view to the project’s URLs:

    from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api-auth/', include('rest_framework.urls')),
]

  • Visit /api-auth/login/ to log in using session-based authentication.

4. Applying Permissions to Views

Permissions control access to API views.

Restrict Access to Authenticated Users

Update the SnippetList view to require authentication:

from rest_framework.permissions import IsAuthenticated
from rest_framework.views import APIView
from rest_framework.response import Response
from .models import Snippet
from .serializers import SnippetSerializer

class SnippetList(APIView):
    permission_classes = [IsAuthenticated]

    def get(self, request):
        snippets = Snippet.objects.all()
        serializer = SnippetSerializer(snippets, many=True)
        return Response(serializer.data)

Explanation:

  • Add permission_classes to specify access rules for the view.
  • In this case, only authenticated users can access the SnippetList endpoint.

5. Customizing Permissions

DRF allows you to define custom permissions by extending BasePermission.

Example: Allow Only Object Owners to Edit or Delete

Create a custom permission in snippets/permissions.py:

from rest_framework.permissions import BasePermission

class IsOwnerOrReadOnly(BasePermission):
    """
    Custom permission to only allow owners of an object to edit or delete it.
    """
    def has_object_permission(self, request, view, obj):
        # Read-only permissions are allowed for any request
        if request.method in ['GET', 'HEAD', 'OPTIONS']:
            return True

        # Write permissions are only allowed to the owner of the snippet
        return obj.owner == request.user

Apply the Custom Permission

Update the SnippetDetail view:

from .permissions import IsOwnerOrReadOnly

class SnippetDetail(APIView):
    permission_classes = [IsOwnerOrReadOnly]

    def get_object(self, pk):
        snippet = Snippet.objects.get(pk=pk)
        self.check_object_permissions(self.request, snippet)
        return snippet

Explanation:

  1. Read-Only Access:
    • All users can view the snippet (GET, HEAD, OPTIONS).
  2. Write Access:
    • Only the snippet owner can edit or delete it.

6. Associating Users with Objects

To enforce owner-based permissions, link snippets to users.

Update the Snippet Model

Modify snippets/models.py to add an owner field:

from django.contrib.auth.models import User

class Snippet(models.Model):
    owner = models.ForeignKey(User, related_name='snippets', on_delete=models.CASCADE)
    # Other fields remain unchanged

Update the Serializer

Include the owner in the serialized data:

class SnippetSerializer(serializers.ModelSerializer):
    owner = serializers.ReadOnlyField(source='owner.username')

    class Meta:
        model = Snippet
        fields = ['id', 'title', 'code', 'linenos', 'language', 'style', 'owner']

Save the Owner in Views

Automatically assign the logged-in user as the owner when creating a snippet:

class SnippetList(APIView):
    permission_classes = [IsAuthenticated]

    def post(self, request):
        serializer = SnippetSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save(owner=request.user)
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

Key Takeaways:

  • Authentication identifies users and is the foundation for securing APIs.
  • Permissions determine what users are allowed to do.
  • DRF provides built-in permissions like IsAuthenticated and supports custom permissions for granular control.
  • Associating users with objects enables owner-based permissions.
