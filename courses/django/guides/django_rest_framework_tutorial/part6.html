Django REST Framework Tutorial: Part 6 - Viewsets and Routers

This tutorial introduces viewsets and routers in Django REST Framework (DRF). Viewsets simplify the process of building APIs by combining logic for handling multiple HTTP methods into a single class, while routers automatically generate URL patterns for these viewsets.

This guide is based on the official Django REST Framework Tutorial: Part 6 - Viewsets and Routers, with expanded explanations and examples.

1. What Are Viewsets?

  • A viewset combines logic for multiple HTTP methods (e.g., GET, POST, PUT, DELETE) into a single class.
  • Instead of writing separate views for listing, creating, retrieving, updating, and deleting, you can use a viewset to handle them all in one place.

2. Refactoring Views to Viewsets

You’ll refactor your existing class-based views into viewsets.

Create Viewsets for Snippets

In snippets/views.py, replace your SnippetList and SnippetDetail views with a SnippetViewSet:

from rest_framework import viewsets
from rest_framework.decorators import action
from rest_framework.response import Response
from rest_framework import renderers
from .models import Snippet
from .serializers import SnippetSerializer
from .permissions import IsOwnerOrReadOnly

class SnippetViewSet(viewsets.ModelViewSet):
    """
    A viewset for viewing and editing snippet instances.
    """
    queryset = Snippet.objects.all()
    serializer_class = SnippetSerializer
    permission_classes = [IsOwnerOrReadOnly]

    @action(detail=True, renderer_classes=[renderers.StaticHTMLRenderer])
    def highlight(self, request, *args, **kwargs):
        snippet = self.get_object()
        return Response(snippet.highlight())

Explanation:

  1. ModelViewSet:
    • Automatically provides implementations for common actions (e.g., list, create, retrieve, update, destroy).
  2. Custom Actions:
    • Use the @action decorator to add custom endpoints, like the highlight method.

Create a Viewset for Users

In snippets/views.py, add a viewset for the User model:

from django.contrib.auth.models import User
from .serializers import UserSerializer

class UserViewSet(viewsets.ReadOnlyModelViewSet):
    """
    A viewset for viewing user instances.
    """
    queryset = User.objects.all()
    serializer_class = UserSerializer
  • ReadOnlyModelViewSet:
    • Provides list and retrieve actions but disallows create, update, and delete.

3. Introducing Routers

DRF’s routers automatically generate URL patterns for viewsets, reducing boilerplate code.

Update snippets/urls.py

Replace your manually defined URLs with a router:

from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import SnippetViewSet, UserViewSet

# Create a router and register viewsets
router = DefaultRouter()
router.register(r'snippets', SnippetViewSet)
router.register(r'users', UserViewSet)

# The API URLs are now determined automatically by the router
urlpatterns = [
    path('', include(router.urls)),
]

Explanation:

  1. DefaultRouter:
    • Automatically generates routes for standard viewset actions (e.g., list, create, retrieve).
    • Includes a default API root view for navigating between endpoints.
  2. Registering Viewsets:
    • Use router.register() to associate a viewset with a URL prefix (e.g., snippets or users).

4. Testing the API

Start the development server and navigate to the API root (http://127.0.0.1:8000/).

Key Features of Router-Generated URLs:

  1. Default API Root:
    • Lists all registered endpoints, such as /snippets/ and /users/.
  2. Detail Endpoints:
    • URLs like /snippets/1/ and /users/1/ allow access to individual resources.
  3. Custom Actions:
    • Custom actions, such as /snippets/1/highlight/, are automatically included.

5. Advantages of Viewsets and Routers

Simplified Code

  • Combining multiple views into a single viewset reduces redundancy and improves readability.

Auto-Generated URLs

  • Routers handle URL routing automatically, eliminating the need to manually define patterns for standard actions.

Extensibility

  • Custom actions allow you to extend viewsets with additional endpoints without cluttering the code.

Key Takeaways:

  • Viewsets encapsulate logic for handling multiple actions (e.g., list, retrieve, create) in a single class.
  • Routers automatically generate URL patterns for viewsets, simplifying API development.
  • DRF’s ModelViewSet and ReadOnlyModelViewSet provide default implementations for standard API actions.
  • Use custom actions to add non-standard endpoints, such as the highlight method.
