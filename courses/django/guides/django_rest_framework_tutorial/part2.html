Django REST Framework Tutorial: Part 2 - Requests and Responses

This tutorial focuses on handling HTTP requests and responses using Django REST Framework (DRF). You’ll learn how to process incoming requests, return appropriate responses, and leverage DRF’s utilities to simplify these interactions.

This guide is based on the official Django REST Framework Tutorial: Part 2 - Requests and Responses, with additional explanations to deepen your understanding.

1. Understanding HTTP Requests in DRF

Django REST Framework extends Django’s standard request and response handling with additional functionality, making it easier to build APIs.

Key Enhancements to HttpRequest:

  1. .data Attribute: DRF extends the HttpRequest object to provide a .data attribute, which parses incoming JSON or form-encoded data.
  2. Flexible Parsing: DRF can handle multiple content types like JSON, form data, and more.

2. Writing a Basic API View

Create a simple API view to handle requests and return responses.

Create a views.py File

In snippets/views.py, define a basic view:

from rest_framework.decorators import api_view
from rest_framework.response import Response
from rest_framework import status

@api_view(['GET', 'POST'])
def snippet_list(request):
    """
    List all code snippets or create a new snippet.
    """
    if request.method == 'GET':
        snippets = Snippet.objects.all()
        serializer = SnippetSerializer(snippets, many=True)
        return Response(serializer.data)

    elif request.method == 'POST':
        serializer = SnippetSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

Explanation:

  1. @api_view: A decorator that defines the allowed HTTP methods (GET, POST, etc.).
  2. GET Requests:
    • Fetch all snippets from the database.
    • Serialize the queryset and return the serialized data in a Response.
  3. POST Requests:
    • Parse and validate incoming data using the serializer.
    • Save the new snippet if the data is valid, or return errors if it’s not.
  4. Response:
    • DRF’s Response object simplifies returning JSON-encoded responses.

3. Creating an API Endpoint

Add a URL pattern for the snippet_list view.

Modify snippets/urls.py

Create a urls.py file for the snippets app:

from django.urls import path
from . import views

urlpatterns = [
    path('snippets/', views.snippet_list, name='snippet-list'),
]

Include App URLs in the Project

Update the project’s urls.py:

from django.urls import include, path

urlpatterns = [
    path('api/', include('snippets.urls')),
]

Start the development server and test the API using tools like:

  • cURL:

  • Postman:

  • Browser:

5. Adding Detail Views

Create another API view to handle a specific snippet using its ID.

Update views.py

@api_view(['GET', 'PUT', 'DELETE'])
def snippet_detail(request, pk):
    """
    Retrieve, update, or delete a snippet instance.
    """
    try:
        snippet = Snippet.objects.get(pk=pk)
    except Snippet.DoesNotExist:
        return Response(status=status.HTTP_404_NOT_FOUND)

    if request.method == 'GET':
        serializer = SnippetSerializer(snippet)
        return Response(serializer.data)

    elif request.method == 'PUT':
        serializer = SnippetSerializer(snippet, data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

    elif request.method == 'DELETE':
        snippet.delete()
        return Response(status=status.HTTP_204_NO_CONTENT)

Explanation:

  1. GET: Retrieve and return a single snippet using its primary key (pk).
  2. PUT: Update a snippet by validating and saving new data.
  3. DELETE: Remove the snippet from the database.

Update urls.py

Add a URL pattern for the detail view:

urlpatterns = [
    path('snippets/', views.snippet_list, name='snippet-list'),
    path('snippets/<int:pk>/', views.snippet_detail, name='snippet-detail'),
]

6. Key Features of Requests and Responses

  • Request Parsing:
    • DRF handles content negotiation automatically (e.g., parsing JSON or form data).
    • Access data using request.data regardless of the content type.
  • Response Handling:
    • Use the Response object to render data in JSON or other formats.
    • Return appropriate HTTP status codes using DRF’s status module.
  • Error Handling:
    • Return detailed error messages and appropriate status codes for client-side errors (e.g., 400 Bad Request).

Key Takeaways:

  • DRF simplifies request and response handling for building APIs.
  • Use @api_view to create function-based API views with specific HTTP methods.
  • Access parsed data using request.data and use the Response class for flexible response formatting.
  • Handle different HTTP methods (GET, POST, PUT, DELETE) with clean, readable code.
  • Test your API endpoints with tools like cURL, Postman, or a browser.
