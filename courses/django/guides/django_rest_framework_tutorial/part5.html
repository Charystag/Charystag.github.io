Django REST Framework Tutorial: Part 5 - Relationships and Hyperlinked APIs

This tutorial covers relationships in Django REST Framework (DRF) and introduces hyperlinked APIs for navigating between related resources. It demonstrates how to serialize relationships between models and represent them using hyperlinks for a more RESTful approach.

This guide is based on the official Django REST Framework Tutorial: Part 5 - Relationships and Hyperlinked APIs, with additional explanations and examples.

1. Understanding Relationships in DRF

In REST APIs, resources often relate to each other:

  • Example: A Snippet belongs to a User.
  • DRF simplifies handling relationships through serializers, which allow you to represent related data using primary keys, nested structures, or hyperlinks.

2. Adding Relationships to the Snippet API

Update the Snippet Model

Ensure the Snippet model has an owner field to establish a relationship with the User model:

from django.contrib.auth.models import User

class Snippet(models.Model):
    owner = models.ForeignKey(User, related_name='snippets', on_delete=models.CASCADE)
    # Other fields remain unchanged

Update the Serializer

Modify SnippetSerializer to include the owner:

from rest_framework import serializers
from .models import Snippet

class SnippetSerializer(serializers.ModelSerializer):
    owner = serializers.ReadOnlyField(source='owner.username')

    class Meta:
        model = Snippet
        fields = ['id', 'title', 'code', 'linenos', 'language', 'style', 'owner']
  • ReadOnlyField: Ensures that the owner field cannot be modified directly and is derived from the username of the related User.

3. Adding a User Serializer

To expose user data and link it to their snippets, create a serializer for the User model.

from django.contrib.auth.models import User
from rest_framework import serializers

class UserSerializer(serializers.ModelSerializer):
    snippets = serializers.PrimaryKeyRelatedField(many=True, queryset=Snippet.objects.all())

    class Meta:
        model = User
        fields = ['id', 'username', 'snippets']
  • PrimaryKeyRelatedField:
    • Represents the related snippets as a list of primary keys.
    • The many=True parameter indicates a one-to-many relationship.

4. Hyperlinking Related Resources

Using HyperlinkedModelSerializer

To make the API more RESTful, replace primary keys with hyperlinks that reference related resources.

Update the SnippetSerializer

class SnippetSerializer(serializers.HyperlinkedModelSerializer):
    owner = serializers.ReadOnlyField(source='owner.username')
    highlight = serializers.HyperlinkedIdentityField(view_name='snippet-highlight', format='html')

    class Meta:
        model = Snippet
        fields = ['url', 'id', 'highlight', 'owner', 'title', 'code', 'linenos', 'language', 'style']
  • HyperlinkedModelSerializer:
    • Adds a url field to represent the hyperlink to the resource.
  • HyperlinkedIdentityField:
    • Creates a hyperlink for a specific view, such as a "highlight" endpoint for snippets.

Update the UserSerializer

class UserSerializer(serializers.HyperlinkedModelSerializer):
    snippets = serializers.HyperlinkedRelatedField(many=True, view_name='snippet-detail', read_only=True)

    class Meta:
        model = User
        fields = ['url', 'id', 'username', 'snippets']
  • HyperlinkedRelatedField:
    • Represents related snippets as hyperlinks to their detail views.

5. Adding a Highlight Endpoint

To add a custom highlight view for snippets, use DRF’s mixins and generics.

Update the Snippet Model

Add a custom highlight method:

from django.utils.html import escape

class Snippet(models.Model):
    # Existing fields remain unchanged

    def highlight(self):
        return escape(self.code)

Add the View

Define a new view for the highlight endpoint:

from rest_framework.decorators import action
from rest_framework.response import Response
from rest_framework import renderers
from rest_framework.generics import get_object_or_404

class SnippetHighlight(generics.GenericAPIView):
    queryset = Snippet.objects.all()
    renderer_classes = [renderers.StaticHTMLRenderer]

    def get(self, request, *args, **kwargs):
        snippet = self.get_object()
        return Response(snippet.highlight())

6. Updating URLs

Update the snippets/urls.py to include the highlight endpoint:

from django.urls import path
from . import views

urlpatterns = [
    path('snippets/', views.SnippetList.as_view(), name='snippet-list'),
    path('snippets/<int:pk>/', views.SnippetDetail.as_view(), name='snippet-detail'),
    path('snippets/<int:pk>/highlight/', views.SnippetHighlight.as_view(), name='snippet-highlight'),
    path('users/', views.UserList.as_view(), name='user-list'),
    path('users/<int:pk>/', views.UserDetail.as_view(), name='user-detail'),
]

7. Testing the Hyperlinked API

Start the development server and navigate to the API in a browser or test with tools like cURL or Postman.

  • Example GET requests: 
    # List all snippets
curl -X GET http://127.0.0.1:8000/snippets/

# Retrieve a specific snippet
curl -X GET http://127.0.0.1:8000/snippets/1/

Key Takeaways:

  • DRF simplifies handling relationships between resources using serializers.
  • Use HyperlinkedModelSerializer to represent related resources as hyperlinks, improving API usability.
  • Add custom endpoints, like the snippet highlight view, for special use cases.
  • Hyperlinked APIs make it easier for clients to navigate and interact with related data.
