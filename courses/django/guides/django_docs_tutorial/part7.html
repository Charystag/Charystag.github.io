Writing Your First Django App, Part 7

This tutorial focuses on customizing Django’s admin interface to improve the usability and efficiency of managing data. The Django admin is a powerful tool for interacting with the database through a user-friendly interface.

1. Registering Models in the Admin Interface

  • By default, only models explicitly registered in admin.py appear in the admin interface.
  • Example: 
    from django.contrib import admin
from .models import Question

admin.site.register(Question)

2. Customizing the Admin Interface

  • Django allows you to customize how models are displayed and edited in the admin interface using ModelAdmin classes.

Customizing Fields

  • Reorder or group fields using the fields or fieldsets attributes in a ModelAdmin class: 
    class QuestionAdmin(admin.ModelAdmin):
    fields = ['pub_date', 'question_text']

Grouping Fields into Sections

  • Use fieldsets to organize fields into collapsible sections: 
    class QuestionAdmin(admin.ModelAdmin):
    fieldsets = [
        (None, {'fields': ['question_text']}),
        ('Date Information', {'fields': ['pub_date'], 'classes': ['collapse']}),
    ]

3. Adding Related Models Inline

  • Display related models (e.g., Choice objects for a Question) directly in the parent model’s admin page using InlineModelAdmin.

  • Example:

    from .models import Choice

class ChoiceInline(admin.TabularInline):
    model = Choice
    extra = 3

class QuestionAdmin(admin.ModelAdmin):
    fieldsets = [
        (None, {'fields': ['question_text']}),
        ('Date Information', {'fields': ['pub_date'], 'classes': ['collapse']}),
    ]
    inlines = [ChoiceInline]

admin.site.register(Question, QuestionAdmin)

  • TabularInline displays choices in a compact table format, while StackedInline uses a stacked layout.

4. Customizing List Display

  • Use the list_display attribute to control the columns displayed on the change list page for a model: 
    class QuestionAdmin(admin.ModelAdmin):
    list_display = ('question_text', 'pub_date', 'was_published_recently')

Adding Filtering and Search

  • Enable filtering by date using list_filter:

    class QuestionAdmin(admin.ModelAdmin):
    list_filter = ['pub_date']

  • Add a search bar for specific fields using search_fields:

    class QuestionAdmin(admin.ModelAdmin):
    search_fields = ['question_text']

5. Testing the Admin Interface

  • Start the development server and log in to the admin interface at http://127.0.0.1:8000/admin/.
  • Experiment with:
    • Adding and editing questions and choices.
    • Using the list display, filters, and search bar.
    • Inline editing of related models.

6. Enhancing Usability

  • Customize admin titles and headers by modifying admin.py: 
    admin.site.site_header = "Polls Administration"
admin.site.site_title = "Polls Admin"
admin.site.index_title = "Welcome to the Polls Administration"

Key Takeaways:

  • The Django admin interface is highly customizable and can be tailored to the needs of the application.
  • Use ModelAdmin classes to adjust field organization, add related models inline, and enhance list views.
  • Filtering and search features improve usability for managing large datasets.
  • Customizing the admin interface allows for efficient data management while maintaining a polished user experience.

This tutorial prepares you to create a streamlined and professional admin interface for your applications. Next, you’ll explore integrating third-party packages and tools to extend Django’s functionality.
