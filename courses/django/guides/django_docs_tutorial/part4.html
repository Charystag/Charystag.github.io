Writing Your First Django App, Part 4

This tutorial focuses on working with forms, handling user input, and processing POST requests. These concepts are essential for creating interactive and dynamic web applications.

1. Adding Forms to Templates

  • Use HTML forms to collect user input for actions like voting.
  • Example: Add a voting form in polls/detail.html: 
    <form action="{% url 'polls:vote' question.id %}" method="post">
    {% csrf_token %}
    {% for choice in question.choice_set.all %}
        <input type="radio" name="choice" id="choice{{ forloop.counter }}" value="{{ choice.id }}">
        <label for="choice{{ forloop.counter }}">{{ choice.choice_text }}</label><br>
    {% endfor %}
    <button type="submit">Vote</button>
</form>
  • Include {% csrf_token %} for protection against Cross-Site Request Forgery (CSRF).

2. Handling POST Requests

  • Add a view to process the form submission:

    from django.http import HttpResponseRedirect
from django.urls import reverse
from .models import Question, Choice

def vote(request, question_id):
    question = get_object_or_404(Question, pk=question_id)
    try:
        selected_choice = question.choice_set.get(pk=request.POST['choice'])
    except (KeyError, Choice.DoesNotExist):
        # Redisplay the question voting form.
        return render(request, 'polls/detail.html', {
            'question': question,
            'error_message': "You didn't select a choice.",
        })
    else:
        selected_choice.votes += 1
        selected_choice.save()
        return HttpResponseRedirect(reverse('polls:results', args=(question.id,)))

  • Key Points:

    • Use request.POST to access form data submitted via POST.
    • Redirect to another view after successful form processing to prevent duplicate submissions.
    • Handle errors gracefully by redisplaying the form with an error message.

3. Displaying Results

  • Create a view to display the voting results:

    from django.shortcuts import get_object_or_404
from .models import Question

def results(request, question_id):
    question = get_object_or_404(Question, pk=question_id)
    return render(request, 'polls/results.html', {'question': question})

  • Add a template for the results page, polls/results.html:

    <h1>{{ question.question_text }}</h1>
<ul>
    {% for choice in question.choice_set.all %}
        <li>{{ choice.choice_text }} -- {{ choice.votes }} vote{{ choice.votes|pluralize }}</li>
    {% endfor %}
</ul>
<a href="{% url 'polls:detail' question.id %}">Vote again</a>

4. Updating URL Patterns

  • Add URL patterns for the results and vote views in polls/urls.py: 
    from django.urls import path
from . import views

app_name = 'polls'
urlpatterns = [
    path('', views.index, name='index'),
    path('<int:question_id>/', views.detail, name='detail'),
    path('<int:question_id>/results/', views.results, name='results'),
    path('<int:question_id>/vote/', views.vote, name='vote'),
]

5. Preventing Double Submissions

  • Redirect to the results page after processing the vote to prevent duplicate submissions when the user refreshes the page:

    from django.http import HttpResponseRedirect
from django.urls import reverse

  • Example:

    return HttpResponseRedirect(reverse('polls:results', args=(question.id,)))

Key Takeaways:

  • Forms allow user interaction and input collection in Django applications.
  • Use POST requests to securely handle form submissions and process data.
  • Protect forms from CSRF attacks using the {% csrf_token %} template tag.
  • Redirect users after processing form data to improve usability and prevent duplicate submissions.
  • Build views to handle voting logic and dynamically display results.
