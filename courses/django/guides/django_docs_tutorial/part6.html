Writing Your First Django App, Part 6

This tutorial focuses on managing and serving static files such as CSS, JavaScript, and images in a Django project. Static files are essential for styling and enhancing the user experience of web applications.

1. What Are Static Files?

  • Static files include CSS, JavaScript, images, or any other files that do not change dynamically with user interaction.
  • Django provides a framework to manage and serve these files efficiently during development and in production.

2. Configuring Static Files

  • Configure the STATIC_URL setting in settings.py to define the URL for accessing static files:

    STATIC_URL = '/static/'

  • For production, set up the STATICFILES_DIRS and STATIC_ROOT settings to manage and collect static files:

    STATICFILES_DIRS = [BASE_DIR / "static"]
STATIC_ROOT = BASE_DIR / "staticfiles"

3. Creating and Serving Static Files

  • Place static files in the static directory of your app. For example:

    polls/
    static/
        polls/
            style.css

  • Reference static files in templates using the {% static %} template tag:

    <link rel="stylesheet" type="text/css" href="{% static 'polls/style.css' %}">

  • Ensure django.contrib.staticfiles is included in the INSTALLED_APPS list.

4. Using Static Files in Development

  • During development, Django automatically serves static files using the STATICFILES_DIRS setting.

Example CSS File:

Create a polls/static/polls/style.css file:

body {
    font-family: Arial, sans-serif;
}

h1 {
    color: blue;
}

Update a Template:

Modify the polls/templates/polls/index.html file to include the CSS:

{% load static %}
<link rel="stylesheet" type="text/css" href="{% static 'polls/style.css' %}">
<h1>Welcome to the Polls App</h1>

5. Managing Images

  • Place images in the static directory, such as polls/static/polls/images/.
  • Use {% static %} to reference images in templates: 
    <img src="{% static 'polls/images/logo.png' %}" alt="Logo">

6. Collecting Static Files for Production

  • Use the collectstatic command to gather all static files into a single location (STATIC_ROOT) for efficient serving: 
    python manage.py collectstatic
  • This is particularly useful when deploying the app to a web server.

7. Debugging Static Files Issues

  • Common issues with static files:
    • Incorrect paths or missing {% static %} in templates.
    • Static files not collected or served correctly in production.
    • Misconfigured STATICFILES_DIRS or STATIC_ROOT.

Key Takeaways:

  • Static files enhance the look and feel of your application.
  • Use the static directory structure and {% static %} template tag to organize and reference files.
  • Configure STATIC_URL and related settings for development and production environments.
  • Use the collectstatic command to prepare static files for deployment.

This tutorial equips you to manage static files effectively, ensuring your app is visually appealing and user-friendly. Next, you'll learn about customizing the Django admin interface to improve data management.
