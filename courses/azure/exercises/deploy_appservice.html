Exercice : Déployez une Application Flask sur Azure App Service via Git

Objectif

Dans cet exercice, vous allez déployer une application Python Flask sur Azure App Service en utilisant :

  1. Le portail Azure pour configurer l’environnement.
  2. Un dépôt Git local pour pousser votre code vers un dépôt distant sur Azure.

Étapes

Étape 1 : Préparation de l'Application Flask

  1. Créez un répertoire pour votre projet local :

    mkdir flask-app
cd flask-app

  2. Créez une application Flask simple :

    • Créez un fichier app.py :

      nano app.py

    • Ajoutez le code suivant :

      from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return "Hello, Azure App Service! Bienvenue dans mon application Flask."

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000)

  3. Créez un environnement virtuel et chargez-le :

    python -m venv .venv
. .venv/bin/activate
Sur windows vous devrez lancer les commandes suivantes: 

py -m venv .venv
.venv\scripts\activate

  1. Installez flask dans l'environnement virtuel:

    pip install flask

  2. Ajoutez un fichier requirements.txt pour les dépendances :

    pip freeze > requirements.txt

  3. Créez un gitignore pour ignorer l'environnement virtuel:

    echo ".venv" > .gitignore

  4. Initialisez un dépôt Git local :

    git init
git add .
git commit -m "Initial commit of Flask app"

Étape 2 : Création d’une Application Web sur Azure App Service

  1. Connectez-vous au portail Azure : https://portal.azure.com.

  2. Créez un App Service Plan :

    • Accédez à "Créer une ressource" > App Service.

    • Remplissez les champs :

      • Nom de l’application : flask-hello-app-<votre_nom> (nom unique).
      • Abonnement : Sélectionnez votre abonnement.
      • Groupe de ressources : Créez-en un nouveau, par exemple FlaskAppServiceGroup.
      • Runtime Stack : Python 3.12 (ou la version disponible la plus proche).
      • Région : westeurope.
      • Plan d'hébergement : Standard (S1) ou Free (F1) pour les tests.

    • Cliquez sur "Créer" pour lancer le déploiement.

  3. Configurez le déploiement via Git :

    • Une fois l’App Service créé, allez dans le menu "Deployment Center" de votre App Service.
    • Sélectionnez Local Git comme source de déploiement.
    • Une URL Git vous sera fournie (exemple : https://flask-hello-app-<votre_nom>.scm.azurewebsites.net:443/flask-hello-app.git).
    • Copiez cette URL, vous en aurez besoin.

Étape 3 : Configuration du Déploiement Local vers Azure

  1. Ajoutez l'URL distante à votre dépôt local :

    git remote add azure <URL_COPIÉE>

    Remplacez <URL_COPIÉE> par l'URL fournie par le portail Azure.

  2. Poussez votre code vers le dépôt distant Azure :

    git push azure master
    • Lors du push, entrez les identifiants Git de déploiement (fournis dans le portail Azure).

Étape 4 : Vérification de l'Application Déployée

  1. Accédez à votre application via l’URL du site web généré par Azure :

    https://flask-hello-app-<votre_nom>.azurewebsites.net

  2. Vérifiez que la page s’affiche correctement :

    Hello, Azure App Service! Bienvenue dans mon application Flask.

Entraînement (Extension non-guidée)

Tâches à Réaliser

  1. Ajoutez une route supplémentaire

    • Créez une route /status qui renvoie un message JSON indiquant que l’application est en ligne.

  2. Ajoutez une variable d’environnement

    • Configurez une variable d’environnement MESSAGE dans Azure App Service pour personnaliser le message d’accueil de la page principale.

  3. Surveillez les journaux de déploiement

    • Utilisez le Portail Azure pour observer les journaux de déploiement et d’exécution.
