Exercice : Déployez un Serveur Web sur une Machine Virtuelle Azure

Objectif

Déployer un serveur web Nginx sur une Machine Virtuelle Azure, héberger une page HTML statique "Hello World", et configurer l’accès réseau.

Étapes

Étape 1 : Création des Ressources Azure

  1. Créez un groupe de ressources dans la région westeurope :

    az group create --name GroupeVMWeb --location "westeurope"

  2. Créez une Machine Virtuelle Ubuntu avec SSH :

    az vm create \
  --resource-group GroupeVMWeb \
  --name VMServeurWeb \
  --image Ubuntu2404 \
  --size Standard_B1s \
  --admin-username azureuser \
  --generate-ssh-keys \
  --location "westeurope"

  3. Configurez le pare-feu pour autoriser le trafic HTTP (port 80) :

    az vm open-port \
  --resource-group GroupeVMWeb \
  --name VMServeurWeb \
  --port 80

Étape 2 : Connexion à la Machine Virtuelle

  1. Connectez-vous à votre VM en utilisant SSH :

    ssh azureuser@<IP_PUBLIQUE>

    Remplacez <IP_PUBLIQUE> par l’adresse IP publique affichée précédemment.

  2. Mettez à jour le système Ubuntu :

    sudo apt update && sudo apt upgrade -y

Étape 3 : Installation de Nginx

  1. Installez Nginx :

    sudo apt install nginx -y

  2. Démarrez Nginx et configurez-le pour démarrer automatiquement :

    sudo systemctl start nginx
sudo systemctl enable nginx

  3. Vérifiez que Nginx fonctionne :

    sudo systemctl status nginx

Étape 4 : Création d'une Page Web Statique

  1. Accédez au répertoire par défaut de Nginx :

    cd /var/www/html

  2. Modifiez le fichier index.html pour afficher "Hello World" :

    sudo bash -c 'echo "<html>
<head><title>Bienvenue</title></head>
<body><h1>Hello World depuis Azure !</h1></body>
</html>" > index.html'

  3. Redémarrez Nginx pour appliquer les modifications :

    sudo systemctl restart nginx

Étape 5 : Accès à la Page Web

  1. Ouvrez un navigateur web et accédez à :

    http://<IP_PUBLIQUE>

  2. Vérifiez que la page affiche :
    "Hello World depuis Azure !"

Entraînement (Extension non-guidée)

Tâches à Réaliser

  1. Personnalisation de la Page Web

    • Modifiez la page index.html pour inclure une structure HTML complète avec des titres, paragraphes, et du style CSS.
    • Affichez dynamiquement la date et l’heure actuelles du serveur.

  2. Ajout d'un Deuxième Serveur Web

    • Installez Apache sur la même VM et configurez-le pour écouter sur le port 8080.
    • Créez une page HTML personnalisée pour Apache.
    • Assurez-vous que les deux serveurs web (Nginx et Apache) fonctionnent simultanément.

  3. Automatisation avec un Script Bash

    • Écrivez un script Bash pour automatiser l’installation de Nginx et la création d’une page HTML personnalisée.
    • Testez votre script en le déployant sur une nouvelle VM.

  4. Nettoyage des Ressources

    • Documentez vos étapes et supprimez les ressources Azure inutilisées.
