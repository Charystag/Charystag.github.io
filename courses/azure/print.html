Exercice : Déployez un Serveur Web sur une Machine Virtuelle Azure

Objectif

Déployer un serveur web Nginx sur une Machine Virtuelle Azure, héberger une page HTML statique "Hello World", et configurer l’accès réseau.

Étapes

Étape 1 : Création des Ressources Azure

  1. Créez un groupe de ressources dans la région westeurope :

    az group create --name GroupeVMWeb --location "westeurope"

  2. Créez une Machine Virtuelle Ubuntu avec SSH :

    az vm create \
  --resource-group GroupeVMWeb \
  --name VMServeurWeb \
  --image Ubuntu2404 \
  --size Standard_B1s \
  --admin-username azureuser \
  --generate-ssh-keys \
  --location "westeurope"

  3. Configurez le pare-feu pour autoriser le trafic HTTP (port 80) :

    az vm open-port \
  --resource-group GroupeVMWeb \
  --name VMServeurWeb \
  --port 80

Étape 2 : Connexion à la Machine Virtuelle

  1. Connectez-vous à votre VM en utilisant SSH :

    ssh azureuser@<IP_PUBLIQUE>

    Remplacez <IP_PUBLIQUE> par l’adresse IP publique affichée précédemment.

  2. Mettez à jour le système Ubuntu :

    sudo apt update && sudo apt upgrade -y

Étape 3 : Installation de Nginx

  1. Installez Nginx :

    sudo apt install nginx -y

  2. Démarrez Nginx et configurez-le pour démarrer automatiquement :

    sudo systemctl start nginx
sudo systemctl enable nginx

  3. Vérifiez que Nginx fonctionne :

    sudo systemctl status nginx

Étape 4 : Création d'une Page Web Statique

  1. Accédez au répertoire par défaut de Nginx :

    cd /var/www/html

  2. Modifiez le fichier index.html pour afficher "Hello World" :

    sudo bash -c 'echo "<html>
<head><title>Bienvenue</title></head>
<body><h1>Hello World depuis Azure !</h1></body>
</html>" > index.html'

  3. Redémarrez Nginx pour appliquer les modifications :

    sudo systemctl restart nginx

Étape 5 : Accès à la Page Web

  1. Ouvrez un navigateur web et accédez à :

    http://<IP_PUBLIQUE>

  2. Vérifiez que la page affiche :
    "Hello World depuis Azure !"

Entraînement (Extension non-guidée)

Tâches à Réaliser

  1. Personnalisation de la Page Web

    • Modifiez la page index.html pour inclure une structure HTML complète avec des titres, paragraphes, et du style CSS.
    • Affichez dynamiquement la date et l’heure actuelles du serveur.

  2. Ajout d'un Deuxième Serveur Web

    • Installez Apache sur la même VM et configurez-le pour écouter sur le port 8080.
    • Créez une page HTML personnalisée pour Apache.
    • Assurez-vous que les deux serveurs web (Nginx et Apache) fonctionnent simultanément.

  3. Automatisation avec un Script Bash

    • Écrivez un script Bash pour automatiser l’installation de Nginx et la création d’une page HTML personnalisée.
    • Testez votre script en le déployant sur une nouvelle VM.

  4. Nettoyage des Ressources

    • Documentez vos étapes et supprimez les ressources Azure inutilisées.

Exercice : Déployez une Application Multi-Tiers avec Azure

Objectif

Déployer une application multi-tiers composée de :

  • Une VM Web hébergeant un serveur Python (Flask).
  • Une VM Base de Données hébergeant une base SQLite (simple à configurer).
    L’objectif est de démontrer la communication entre les deux serveurs.

Prérequis

  • Compte Azure avec droits d’administration.
  • Azure CLI installé ou utilisation d’Azure Cloud Shell.
  • Connaissance de base des commandes Linux.

Étapes

Étape 1 : Création des Ressources Azure

  1. Créez un groupe de ressources dans Azure CloudShell :

    az group create --name GroupeMultiTiers --location "westeurope"

  2. Créez deux machines virtuelles Ubuntu :

    • VM1 : Serveur Web
    • VM2 : Serveur Base de Données

    Pour la VM Web (WebServerVM) :

    az vm create \
  --resource-group GroupeMultiTiers \
  --name WebServerVM \
  --image UbuntuLTS \
  --size Standard_B1s \
  --admin-username azureuser \
  --generate-ssh-keys \
  --location "westeurope"

    Pour la VM Base de Données (DBServerVM) :

    az vm create \
  --resource-group GroupeMultiTiers \
  --name DBServerVM \
  --image UbuntuLTS \
  --size Standard_B1s \
  --admin-username azureuser \
  --generate-ssh-keys \
  --location "westeurope"

  3. Configurez les règles de pare-feu pour autoriser :

    • HTTP (port 80) sur la VM Web : 
      az vm open-port --resource-group GroupeMultiTiers --name WebServerVM --port 80
    • SSH (port 22) sur les deux VMs (déjà configuré par défaut).
    • Port 5000 pour le serveur de base de données : 
      az vm open-port --resource-group GroupeMultiTiers --name DBServerVM --port 5000

Étape 2 : Configuration du Serveur Base de Données (SQLite)

  1. Connectez-vous à la VM Base de Données :

    ssh azureuser@<IP_PUBLIQUE_DBServerVM>

  2. Installez Python et SQLite :

    sudo apt update
sudo apt install python3 python3-pip sqlite3 -y

  3. Créez une base de données SQLite :

    mkdir ~/db
cd ~/db
sqlite3 demo.db "CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT);"
sqlite3 demo.db "INSERT INTO users (name) VALUES ('Alice'), ('Bob');"

  4. Démarrez un serveur Flask pour accéder à la base de données :
    Créez un fichier db_server.py :

    nano db_server.py

    Contenu de db_server.py :

    import sqlite3
from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/users')
def get_users():
    conn = sqlite3.connect('/home/azureuser/db/demo.db')
    cursor = conn.cursor()
    cursor.execute("SELECT * FROM users")
    users = [{"id": row[0], "name": row[1]} for row in cursor.fetchall()]
    conn.close()
    return jsonify(users)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

  5. Installez Flask et lancez le serveur :

    pip3 install flask
python3 db_server.py

Étape 3 : Configuration du Serveur Web

  1. Connectez-vous à la VM Web :

    ssh azureuser@<IP_PUBLIQUE_WebServerVM>

  2. Installez Python et Flask :

    sudo apt update
sudo apt install python3 python3-pip -y
pip3 install flask requests

  3. Créez un serveur Flask pour accéder aux données de la VM Base de Données :

    nano web_server.py

    Contenu de web_server.py :

    import requests
from flask import Flask, jsonify

app = Flask(__name__)

DB_SERVER_IP = "<IP_PRIVÉE_DBServerVM>"

@app.route('/')
def home():
    return "Bienvenue sur le serveur web ! Accédez aux utilisateurs via /users"

@app.route('/users')
def get_users():
    try:
        response = requests.get(f"http://{DB_SERVER_IP}:5000/users")
        return jsonify(response.json())
    except Exception as e:
        return jsonify({"error": str(e)})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=80)

  4. Récupérez l’adresse IP privée de la VM Base de Données :

    az vm show -d --resource-group GroupeMultiTiers --name DBServerVM --query privateIps -o tsv

  5. Remplacez <IP_PRIVÉE_DBServerVM> dans web_server.py avec l’adresse récupérée.

  6. Lancez le serveur Flask :

    sudo python3 web_server.py

Étape 4 : Test de Communication entre les VMs

  1. Accédez à la VM Web via son IP publique dans un navigateur :

    http://<IP_PUBLIQUE_WebServerVM>

  2. Accédez aux données de la base de données via la route /users :

    http://<IP_PUBLIQUE_WebServerVM>/users
    • Vous devriez voir les utilisateurs "Alice" et "Bob" provenant de la base de données SQLite sur la VM Base de Données.

Entraînement (Extension non-guidée)

  1. Ajout de Route /add_user
    Ajoutez une fonctionnalité pour insérer dynamiquement des utilisateurs via une route POST.

  2. Automatisation

    • Écrivez un script Bash pour configurer automatiquement les deux serveurs.

Tutoriel : Création et gestion d’un groupe de ressources sur Azure

Un groupe de ressources dans Azure est une unité logique qui regroupe des ressources cloud comme des machines virtuelles, des bases de données, des réseaux, etc. Cette organisation facilite leur gestion, leur supervision et leur contrôle.

Qu’est-ce qu’un groupe de ressources dans Azure ?

Un groupe de ressources Azure est un conteneur logique qui contient des ressources liées. Ces ressources peuvent inclure :

  • Machines virtuelles,
  • Comptes de stockage,
  • Réseaux virtuels,
  • Bases de données.

Avantages :

  • Organisation : Regroupez des ressources liées à un même projet ou environnement.
  • Gestion simplifiée : Appliquez des stratégies ou surveillez toutes les ressources en une seule action.
  • Coûts et budgets : Analysez les dépenses par projet ou application.

Étape 1 : Accéder au Portail Azure

  1. Rendez-vous sur le Portail Azure.
  2. Connectez-vous avec vos identifiants.

Étape 2 : Créer un groupe de ressources

  1. Naviguer vers la section des groupes de ressources :

    • Dans le menu latéral gauche, cliquez sur Groupes de ressources.
    • Si ce menu n’est pas visible, recherchez "Groupes de ressources" dans la barre de recherche en haut.

  2. Créer un nouveau groupe de ressources :

    • Cliquez sur le bouton + Créer.
    • Fournissez les informations nécessaires :
      • Abonnement : Choisissez l’abonnement Azure à utiliser.
      • Nom du groupe de ressources : Donnez un nom explicite (ex. : Groupe-ProjetXYZ).
      • Région : Choisissez la région Azure où les métadonnées du groupe seront stockées.

        Remarque : Les ressources dans ce groupe peuvent être dans d’autres régions.

  3. Vérifiez et créez :

    • Cliquez sur Vérifier + Créer.
    • Une fois la validation effectuée, cliquez sur Créer.

Étape 3 : Ajouter des ressources à un groupe de ressources

  1. Depuis le groupe de ressources :

    • Une fois créé, cliquez sur votre groupe dans la liste des groupes de ressources.
    • Cliquez sur + Ajouter pour créer une nouvelle ressource dans ce groupe.

  2. Depuis une ressource existante :

    • Lorsque vous créez une ressource (par exemple, une machine virtuelle), sélectionnez le groupe de ressources lors de la configuration.

Étape 4 : Gérer un groupe de ressources

4.1 Modifier les paramètres :

  • Ouvrez le groupe de ressources.
  • Accédez à l'onglet Paramètres pour :
    • Ajouter des étiquettes (tags) pour organiser vos ressources.
    • Consulter les informations sur le groupe (région, abonnement, etc.).

4.2 Supprimer des ressources :

  • Dans le groupe de ressources, sélectionnez une ressource.
  • Cliquez sur Supprimer pour la supprimer individuellement.

4.3 Supprimer le groupe de ressources :

  • Attention : La suppression d’un groupe de ressources entraîne la suppression de toutes les ressources qu’il contient.
    • Ouvrez le groupe.
    • Cliquez sur Supprimer le groupe de ressources.
    • Tapez le nom du groupe pour confirmer l’action.

4.4 Contrôler les coûts :

  • Accédez à Analyse des coûts depuis le groupe de ressources pour suivre les dépenses liées aux ressources.

Étape 5 : Appliquer des stratégies ou gérer l'accès

5.1 Appliquer une stratégie (Azure Policy) :

  • Dans le groupe, allez à Stratégie et attribuez des règles :
    • Exemples : restreindre les régions pour les ressources ou limiter certains types de ressources.

5.2 Gérer l'accès (RBAC - Contrôle d’accès basé sur les rôles) :

  • Cliquez sur Contrôle d’accès (IAM) dans le menu du groupe.
  • Ajoutez des utilisateurs ou groupes avec des rôles spécifiques (Lecteur, Contributeur, Propriétaire).

Bonnes pratiques pour la gestion des groupes de ressources

  1. Nommez vos groupes de manière claire :

    • Incluez des informations comme le projet, l’environnement (Dev/Test/Prod), ou l’équipe.
    • Exemple : ProjetX-Dev-Resources.

  2. Utilisez des étiquettes (Tags) :

    • Les étiquettes permettent d’ajouter des métadonnées personnalisées aux groupes et ressources (ex. : Equipe:Marketing, Projet:SiteWeb).

  3. Centralisez les ressources liées :

    • Regroupez les ressources qui ont un cycle de vie similaire ou qui appartiennent à une même application.

  4. Surveillez régulièrement :

    • Utilisez Azure Monitor pour suivre les performances et l’état des ressources du groupe.

Tutoriel : Déploiement et gestion des étiquettes (tags) pour une meilleure organisation dans Azure

Les étiquettes (tags) dans Microsoft Azure sont des paires clé-valeur que vous pouvez appliquer à des ressources, des groupes de ressources, et des abonnements pour améliorer leur organisation et leur gestion. Ce tutoriel vous guide à travers le déploiement et la gestion des étiquettes dans Azure.

Pourquoi utiliser les étiquettes dans Azure ?

  1. Organisation : Classez et identifiez les ressources par projet, équipe, environnement (prod/dev/test), ou toute autre catégorie.
  2. Analyse des coûts : Attribuez des coûts à des projets ou départements spécifiques.
  3. Automatisation : Simplifiez les opérations avec des scripts et des stratégies basés sur les étiquettes.
  4. Gestion centralisée : Filtrez et recherchez rapidement des ressources dans le portail Azure.

Étape 1 : Comprendre les étiquettes

Une étiquette se compose de deux parties :

  • Clé : Le nom ou la catégorie de la étiquette (ex. : Projet, Environnement, Equipe).
  • Valeur : L’information associée à la clé (ex. : SiteWeb, Production, Marketing).

Exemple :
Clé : Projet → Valeur : SiteWeb

Étape 2 : Ajouter des étiquettes à une ressource

  1. Depuis le Portail Azure :

    • Accédez à une ressource (par exemple, une machine virtuelle ou un compte de stockage).
    • Cliquez sur Étiquettes dans le menu latéral.
    • Ajoutez des paires clé-valeur :
      • Exemple : Clé : Environnement → Valeur : Production
    • Cliquez sur Enregistrer.

  2. Lors de la création d'une ressource :

    • Lors de la configuration d’une nouvelle ressource, cherchez l’onglet ou la section Étiquettes.
    • Ajoutez les étiquettes nécessaires avant de finaliser la création.

Étape 3 : Ajouter des étiquettes à plusieurs ressources simultanément

  1. Utilisation du portail Azure :

    • Dans la barre de recherche du portail, tapez Étiquettes et sélectionnez l'option.
    • Cliquez sur Ajouter une étiquette à des ressources.
    • Sélectionnez plusieurs ressources dans la liste.
    • Ajoutez les étiquettes souhaitées (Clé/valeur).
    • Cliquez sur Appliquer.

  2. Avec Azure PowerShell : Utilisez PowerShell pour automatiser l’application de étiquettes :

    Set-AzResource -ResourceId "/subscriptions/<SubscriptionID>/resourceGroups/<ResourceGroupName>/providers/<ResourceType>/<ResourceName>" `
-Tag @{ "Projet" = "SiteWeb"; "Environnement" = "Production" }

  3. Avec Azure CLI : Vous pouvez également utiliser Azure CLI :

    az resource tag --ids "/subscriptions/<SubscriptionID>/resourceGroups/<ResourceGroupName>/providers/<ResourceType>/<ResourceName>" \
--tags Projet=SiteWeb Environnement=Production

Étape 4 : Gérer et modifier les étiquettes

  1. Modifier des étiquettes :

    • Accédez à une ressource ou un groupe de ressources.
    • Cliquez sur Étiquettes, modifiez les paires clé-valeur existantes ou ajoutez-en de nouvelles.
    • Enregistrez vos modifications.

  2. Supprimer des étiquettes :

    • Supprimez une clé ou une valeur dans la section Étiquettes.
    • Cliquez sur Enregistrer pour confirmer.

Étape 5 : Utiliser Azure Policy pour appliquer des étiquettes automatiquement

Azure Policy permet de définir des règles pour garantir que toutes les ressources ont les étiquettes nécessaires. Par exemple, vous pouvez exiger que toutes les ressources dans un groupe de ressources aient une étiquette Environnement.

  1. Créer une stratégie pour les étiquettes :

    • Accédez à Azure Policy dans le portail.
    • Cliquez sur Définitions > + Ajouter une définition personnalisée.
    • Utilisez une définition intégrée ou personnalisez une règle. Exemple d'une définition intégrée : Ajouter une étiquette à une ressource lors de son création.
    • Spécifiez les détails de la stratégie (clé/valeur).

  2. Affecter la stratégie :

    • Allez dans Assignations dans Azure Policy.
    • Affectez la stratégie à un abonnement ou un groupe de ressources.

  3. Automatiser les corrections :

    • Configurez la stratégie pour appliquer automatiquement les étiquettes manquantes.

Étape 6 : Surveiller les ressources par étiquettes

  1. Filtrer les ressources par étiquettes :

    • Accédez à la section Tous les services > Ressources.
    • Cliquez sur Filtrer et choisissez une étiquette (ex. : Projet=SiteWeb).

  2. Analyse des coûts par étiquettes :

    • Accédez à Analyse des coûts.
    • Configurez un rapport de coût basé sur les étiquettes (ex. : afficher les coûts par environnement).

Bonnes pratiques pour les étiquettes

  1. Standardisez les clés : Utilisez des noms cohérents et explicites (Projet, Environnement, Equipe) pour éviter les doublons.
  2. Automatisez l’application : Utilisez Azure Policy pour éviter les oublis.
  3. Limitez les clés personnalisées : Utilisez des clés communes pour simplifier la gestion.
  4. Surveillez et nettoyez régulièrement : Vérifiez les ressources avec des étiquettes incomplètes ou incorrectes.

Tutoriel : Déployer une machine virtuelle Windows/Linux sur Azure

Les machines virtuelles (VM) dans Azure permettent de créer et de gérer des serveurs virtuels exécutant Windows ou Linux. Ce tutoriel vous guidera à travers les étapes de déploiement d'une VM depuis le portail Azure.

Étape 1 : Accéder au portail Azure

  1. Connectez-vous à votre compte sur le portail Azure.
  2. Dans la barre de recherche, tapez "Machines virtuelles" et cliquez sur le service correspondant.

Étape 2 : Créer une nouvelle machine virtuelle

  1. Cliquer sur "Créer" :

    • Dans la section Machines virtuelles, cliquez sur + Créer puis Machine virtuelle Azure.

  2. Configuration de base : Remplissez les champs suivants :

    • Abonnement : Sélectionnez l’abonnement Azure à utiliser.
    • Groupe de ressources : Choisissez un groupe existant ou créez-en un nouveau.
    • Nom de la machine virtuelle : Entrez un nom explicite pour la VM.
    • Région : Sélectionnez la région Azure où vous souhaitez héberger la VM.
    • Options de disponibilité : Configurez la redondance pour garantir la haute disponibilité (facultatif).

  3. Système d’exploitation :

    • Sélectionnez le système d’exploitation :
      • Windows Server : Ex. : Windows Server 2019.
      • Linux : Ex. : Ubuntu Server 20.04 LTS.

  4. Taille de la machine virtuelle :

    • Cliquez sur Changer de taille et sélectionnez une taille en fonction des besoins (CPU, RAM).
    • Exemples courants :
      • B1ms pour des tests de base.
      • D2s_v3 pour des charges de travail standard.

  5. Options d'authentification :

    • Pour Windows : Sélectionnez l’authentification par nom d’utilisateur et mot de passe ou une clé SSH (facultatif).
    • Pour Linux : Utilisez une clé SSH (recommandée) ou un mot de passe.
      • Si vous choisissez une clé SSH, générez-en une via Azure ou utilisez votre propre clé.

Étape 3 : Configuration des disques

  1. Disque du système d’exploitation :

    • Sélectionnez un type de disque : SSD Premium, SSD Standard ou HDD Standard.
    • SSD Premium est recommandé pour des performances élevées.

  2. Disques de données (facultatif) :

    • Cliquez sur Créer et attacher un nouveau disque si vous avez besoin de stockage supplémentaire.

Étape 4 : Configuration réseau

  1. Réseau virtuel :

    • Azure configure automatiquement un réseau virtuel par défaut. Vous pouvez également créer un réseau personnalisé.

  2. Sous-réseau :

    • Vérifiez que le sous-réseau par défaut est sélectionné.

  3. Adresse IP publique :

    • Activez l'adresse IP publique pour permettre l'accès à distance.

  4. Port d’entrée :

    • Pour Windows, ouvrez le port RDP (3389) pour l'accès à distance.
    • Pour Linux, ouvrez le port SSH (22).

Étape 5 : Vérification et déploiement

  1. Options avancées (facultatif) :

    • Configurez des extensions, des scripts de démarrage ou d'autres paramètres avancés selon vos besoins.

  2. Revue et création :

    • Cliquez sur Vérifier + créer.
    • Azure validera vos configurations.
    • Cliquez sur Créer pour démarrer le déploiement.

Étape 6 : Connexion à la machine virtuelle

  1. Pour Windows :

    • Accédez à la VM dans le portail Azure.
    • Cliquez sur Se connecter > RDP.
    • Téléchargez le fichier RDP, ouvrez-le avec votre client RDP, et connectez-vous avec vos identifiants.

  2. Pour Linux :

    • Accédez à la VM dans le portail Azure.
    • Cliquez sur Se connecter > SSH.
    • Suivez les instructions pour utiliser la clé SSH et la commande appropriée : 
      ssh <NomUtilisateur>@<AdresseIP>

Étape 7 : Gérer la machine virtuelle

  1. Arrêter/Démarrer :

    • Vous pouvez arrêter ou redémarrer la VM depuis le portail.

  2. Superviser les performances :

    • Allez dans l’onglet Surveillance pour consulter les métriques (CPU, RAM, etc.).

  3. Sauvegardes :

    • Configurez des sauvegardes dans l’onglet Sauvegarde pour éviter la perte de données.

  4. Sécurité :

    • Vérifiez régulièrement les règles de pare-feu.
    • Utilisez Azure Bastion pour une connexion plus sécurisée.

Bonnes pratiques

  1. Fermer les ports inutilisés :

    • Ne laissez ouverts que les ports nécessaires (3389 pour RDP, 22 pour SSH).

  2. Automatisez :

    • Utilisez des outils comme Azure CLI, PowerShell ou des modèles ARM pour déployer des VMs plus rapidement.

  3. Gérez les coûts :

    • Arrêtez les VMs inutilisées pour réduire les coûts.
    • Surveillez les dépenses avec Azure Cost Management.

  4. Maintenez vos VMs à jour :

    • Appliquez les mises à jour système pour améliorer la sécurité et les performances.

Tutoriel : Connexion à une Machine Virtuelle (VM) avec RDP ou SSH sur Azure

Se connecter à une machine virtuelle sur Azure permet d’accéder à son interface pour la configurer, exécuter des applications, ou effectuer des maintenances. Ce guide vous explique comment se connecter à une VM, que ce soit avec Remote Desktop Protocol (RDP) pour Windows ou Secure Shell (SSH) pour Linux.

Prérequis pour la connexion

  1. Pour RDP (Windows) :

    • Une VM Windows déployée.
    • Le port 3389 doit être ouvert dans les paramètres réseau de la VM.
    • Vous devez disposer des identifiants de connexion (nom d'utilisateur et mot de passe).

  2. Pour SSH (Linux) :

    • Une VM Linux déployée.
    • Le port 22 doit être ouvert dans les paramètres réseau de la VM.
    • Vous devez disposer des clés SSH ou des identifiants utilisateur/mot de passe.

  3. Adresse IP publique :

    • La VM doit avoir une adresse IP publique pour une connexion externe.

Étape 1 : Vérifier les configurations réseau de la VM

Avant de vous connecter :

  1. Accédez à votre VM dans le portail Azure.
  2. Allez dans l’onglet Réseau :
    • Vérifiez que l'adresse IP publique est activée.

Tutoriel : Utiliser le protocole SCP pour transférer des fichiers sur une VM Linux dans Azure

SCP (Secure Copy Protocol) est un outil puissant pour transférer des fichiers entre un système local et une machine virtuelle Linux en utilisant le protocole SSH. Ce guide vous explique comment utiliser SCP pour transférer des fichiers sur une VM Linux hébergée sur Azure.

Prérequis

  1. Une machine virtuelle Linux déployée sur Azure avec :

    • Un port SSH (22) ouvert dans les règles du groupe de sécurité réseau (NSG).
    • Une clé SSH ou un mot de passe pour vous connecter à la VM.

  2. Une adresse IP publique associée à la VM.

  3. SCP installé sur votre système local :

    • SCP est inclus par défaut sur les systèmes Linux et MacOS.
    • Pour Windows, utilisez un outil comme PuTTY (avec pscp) ou installez OpenSSH via PowerShell.

Étape 1 : Vérifiez la connectivité SSH avec la VM

  1. Récupérez l'adresse IP publique de la VM :

    • Dans le portail Azure, allez dans Machines virtuelles > Votre VM > Vue d’ensemble.
    • Notez l'adresse IP publique.

  2. Testez la connexion SSH depuis votre terminal :

    ssh <NomUtilisateur>@<AdresseIP>

    Exemple :

    ssh azureuser@20.50.60.70

    Si la connexion est réussie, vous êtes prêt à utiliser SCP.

Étape 2 : Transférer des fichiers vers la VM avec SCP

Transfert d’un fichier unique

Utilisez la commande suivante pour copier un fichier local vers votre VM Linux :

scp <CheminLocalDuFichier> <NomUtilisateur>@<AdresseIP>:<CheminCible>
  • : Chemin complet du fichier à transférer.
  • : Nom d'utilisateur pour la connexion SSH (souvent azureuser).
  • : Adresse IP publique de la VM.
  • : Chemin où le fichier sera stocké sur la VM.

Exemple :

Pour copier un fichier nommé rapport.txt dans le répertoire /home/azureuser de la VM :

scp ~/Documents/rapport.txt azureuser@20.50.60.70:/home/azureuser/

Transfert de plusieurs fichiers ou répertoires

Pour transférer un répertoire entier vers la VM, utilisez l'option -r (récursif) :

scp -r <CheminLocalDuDossier> <NomUtilisateur>@<AdresseIP>:<CheminCible>

Exemple :

Pour copier un dossier projet vers /home/azureuser/ sur la VM :

scp -r ~/projet azureuser@20.50.60.70:/home/azureuser/

Étape 3 : Télécharger des fichiers depuis la VM

Pour copier des fichiers ou répertoires de la VM vers votre machine locale, inversez les chemins dans la commande SCP.

Téléchargement d’un fichier unique

scp <NomUtilisateur>@<AdresseIP>:<CheminDuFichierSurLaVM> <CheminLocalDeDestination>

Exemple :

Pour copier un fichier rapport.txt depuis la VM vers le répertoire local ~/Documents :

scp azureuser@20.50.60.70:/home/azureuser/rapport.txt ~/Documents/

Téléchargement d’un répertoire

Utilisez l'option -r pour copier un répertoire entier :

scp -r <NomUtilisateur>@<AdresseIP>:<CheminDuDossierSurLaVM> <CheminLocalDeDestination>

Exemple :

Pour copier un dossier projet depuis /home/azureuser de la VM vers ~/Documents sur votre machine locale :

scp -r azureuser@20.50.60.70:/home/azureuser/projet ~/Documents/

Étape 4 : Résolution des problèmes courants

  1. Erreur "Permission denied" :

    • Vérifiez que vous utilisez le bon nom d'utilisateur et que votre clé SSH ou mot de passe est correct.
    • Assurez-vous que le port 22 est ouvert dans les règles NSG de la VM.

  2. Connexion impossible (Timeout) :

    • Vérifiez que votre adresse IP est autorisée dans les règles du pare-feu réseau Azure.
    • Confirmez que la VM est en cours d'exécution.

  3. Accès refusé au chemin cible sur la VM :

    • Assurez-vous que l'utilisateur a les permissions nécessaires pour écrire dans le répertoire cible.

Étape 5 : Automatisation avec des scripts

Vous pouvez automatiser les transferts de fichiers avec un script Shell.

Exemple de script pour transférer des fichiers vers la VM :

#!/bin/bash

# Variables
USER="azureuser"
IP="20.50.60.70"
LOCAL_PATH="/path/to/local/file"
REMOTE_PATH="/path/to/remote/destination"

# Transférer un fichier
scp $LOCAL_PATH $USER@$IP:$REMOTE_PATH

Exemple de script pour sauvegarder des fichiers depuis la VM :

#!/bin/bash

# Variables
USER="azureuser"
IP="20.50.60.70"
REMOTE_PATH="/path/to/remote/file"
LOCAL_PATH="/path/to/local/destination"

# Télécharger un fichier
scp $USER@$IP:$REMOTE_PATH $LOCAL_PATH

Bonnes pratiques pour le transfert SCP

  1. Utilisez des clés SSH :
    • Préférez les clés SSH au lieu des mots de passe pour des connexions plus sécurisées.
  2. Réduisez l’exposition publique :
    • Utilisez des règles NSG pour limiter l’accès SSH à des adresses IP spécifiques.
  3. Compression :
    • Ajoutez l'option -C pour compresser les fichiers pendant le transfert, améliorant la vitesse.
    scp -C <CheminLocalDuFichier> <NomUtilisateur>@<AdresseIP>:<CheminCible>

Tutoriel : Gérer les sauvegardes et les snapshots des Machines Virtuelles (VMs) sur Azure

La gestion des sauvegardes et des snapshots des machines virtuelles (VMs) dans Azure est essentielle pour garantir la continuité des opérations et éviter les pertes de données en cas de panne. Ce tutoriel explique comment configurer et gérer les sauvegardes et les snapshots des VMs.

1. Différence entre sauvegardes et snapshots

Sauvegardes

  • Description : Les sauvegardes Azure (via le service Azure Backup) permettent de créer des copies cohérentes de vos VMs, incluant le système d'exploitation, les disques, et les données.
  • Utilisation : Restauration complète ou partielle en cas de panne, de suppression accidentelle ou de corruption de données.
  • Avantages :
    • Gestion centralisée.
    • Protection automatisée avec des stratégies de rétention.
    • Chiffrement des sauvegardes.

Snapshots

  • Description : Un snapshot est une copie ponctuelle d’un disque VM spécifique (généralement un disque OS ou un disque de données).
  • Utilisation : Revenir rapidement à un état précédent, cloner des disques, ou tester des modifications.
  • Avantages :
    • Plus rapide à créer que les sauvegardes.
    • Utile pour des sauvegardes temporaires ou des tests.

2. Gérer les sauvegardes d'une VM avec Azure Backup

Étape 1 : Configurer une sauvegarde pour une VM

  1. Accéder à la VM :

    • Dans le portail Azure, recherchez et sélectionnez votre machine virtuelle.

  2. Configurer Azure Backup :

    • Dans le menu latéral, cliquez sur Sauvegarde.
    • Sélectionnez un Coffre Recovery Services existant ou créez-en un :
      • Donnez un nom au coffre.
      • Sélectionnez une région et un abonnement.

  3. Configurer la stratégie de sauvegarde :

    • Azure propose une stratégie par défaut :
      • Sauvegarde quotidienne.
      • Conservation des sauvegardes pendant 30 jours.
    • Vous pouvez personnaliser les paramètres :
      • Fréquence (quotidienne/hebdomadaire).
      • Durée de conservation (jours, semaines, mois, années).

  4. Activer la sauvegarde :

    • Cliquez sur Activer la sauvegarde pour démarrer la protection de la VM.

Étape 2 : Restauration depuis une sauvegarde

  1. Accéder au Coffre Recovery Services :

    • Allez dans Coffres Recovery Services depuis le portail Azure.
    • Sélectionnez le coffre contenant la sauvegarde.

  2. Choisir l'élément à restaurer :

    • Cliquez sur Articles sauvegardés.
    • Sélectionnez la VM et choisissez un point de restauration.

  3. Options de restauration :

    • Créer une nouvelle VM : Crée une nouvelle VM à partir de la sauvegarde.
    • Restauration des disques : Restaure uniquement les disques, que vous pouvez attacher à une VM existante.

3. Gérer les snapshots des disques VM

Étape 1 : Créer un snapshot

  1. Accéder au disque de la VM :

    • Dans le portail Azure, ouvrez la VM.
    • Cliquez sur Disques dans le menu.
    • Sélectionnez le disque que vous voulez snapshotter (ex. : disque OS ou disque de données).

  2. Créer un snapshot :

    • Dans l'onglet Vue d'ensemble du disque, cliquez sur Créer un snapshot.
    • Configurez les paramètres :
      • Nom : Donnez un nom explicite (ex. : Snapshot-Disque-Prod).
      • Type de snapshot : Sélectionnez Snapshot complet (copie complète) ou Snapshot incrémental (plus rapide et moins coûteux).
      • Compte de stockage : Spécifiez où stocker le snapshot.

  3. Créer :

    • Cliquez sur Créer pour démarrer la création du snapshot.

Étape 2 : Utiliser un snapshot

  1. Créer un nouveau disque depuis un snapshot :

    • Allez dans le service Snapshots dans le portail Azure.
    • Sélectionnez le snapshot souhaité.
    • Cliquez sur Créer un disque et configurez les options (type, taille, etc.).
    • Le disque créé peut ensuite être attaché à une VM.

  2. Remplacer un disque existant :

    • Arrêtez la VM.
    • Détachez le disque existant.
    • Attachez le disque restauré à partir du snapshot.

Étape 3 : Supprimer un snapshot

  1. Accéder au service Snapshots :

    • Recherchez Snapshots dans le portail Azure.
    • Sélectionnez le snapshot que vous souhaitez supprimer.

  2. Supprimer :

    • Cliquez sur Supprimer pour libérer de l'espace de stockage.

4. Bonnes pratiques pour les sauvegardes et snapshots

Sauvegardes

  1. Automatisez les sauvegardes :
    • Utilisez des stratégies pour garantir que toutes les VMs critiques sont protégées.
  2. Surveillez l'état des sauvegardes :
    • Configurez des alertes pour les échecs de sauvegarde.
  3. Testez régulièrement les restaurations :
    • Assurez-vous que les points de restauration sont valides et utilisables.

Snapshots

  1. Utilisez des snapshots pour des modifications rapides :
    • Avant une mise à jour majeure ou des tests.
  2. Nettoyez régulièrement :
    • Supprimez les snapshots obsolètes pour économiser les coûts de stockage.
  3. Combinez snapshots et sauvegardes :
    • Utilisez les sauvegardes pour des restaurations complètes et les snapshots pour des interventions rapides.

Tutoriel : Configurer un compte de stockage Azure

Les comptes de stockage Azure sont des solutions flexibles pour stocker des données dans le cloud. Ils permettent de gérer des objets (blobs), des fichiers, des disques, des tables et des files d'attente. Ce tutoriel explique comment configurer et gérer un compte de stockage Azure.

Étape 1 : Comprendre les types de stockage Azure

Avant de commencer, comprenez les principaux types de stockage offerts par Azure :

  1. Blob Storage : Stockage d’objets pour les données non structurées (images, vidéos, sauvegardes).
  2. File Storage : Partages de fichiers accessibles via SMB ou NFS.
  3. Queue Storage : Files d'attente de messages pour la communication asynchrone.
  4. Table Storage : Stockage NoSQL pour les données structurées.

Étape 2 : Créer un compte de stockage Azure

  • Connectez-vous au portail Azure.
  • Recherchez Comptes de stockage dans la barre de recherche et cliquez sur l’option correspondante.

  • Remplissez les informations nécessaires :

Onglet "De base"

  • Abonnement : Sélectionnez l'abonnement Azure à utiliser.
  • Groupe de ressources : Choisissez un groupe existant ou créez-en un nouveau.
  • Nom du compte de stockage : Donnez un nom unique (entre 3 et 24 caractères, uniquement alphanumériques).
  • Région : Sélectionnez la région Azure où vous souhaitez héberger le compte.
  • Performance :
    • Standard : Idéal pour des applications courantes avec des besoins modérés en performance.
    • Premium : Pour des applications nécessitant des performances élevées (par exemple, bases de données).
  • Redondance :
    • LRS (Locally Redundant Storage) : Redondance dans une seule région.
    • GRS (Geo-Redundant Storage) : Redondance géographique pour la haute disponibilité.

Onglets supplémentaires (facultatif)

  • Réseau : Configurez l’accès privé ou public.

  • Sécurité : Activez le chiffrement avec des clés gérées par le client ou Azure.

  • Avancé : Configurez des fonctionnalités telles que le stockage hiérarchisé.

  • Une fois tous les champs remplis, cliquez sur Vérifier + Créer puis sur Créer.

Étape 3 : Configurer le compte de stockage

1. Accéder au compte de stockage

  • Une fois créé, cliquez sur le compte dans la liste des Comptes de stockage.

2. Configurer des services spécifiques

  • Blob Storage :
    • Allez dans l'onglet Conteneurs.
    • Cliquez sur + Conteneur pour créer un nouveau conteneur.
    • Définissez les permissions d'accès (Public/Privé).
  • File Storage :
    • Accédez à Partages de fichiers.
    • Cliquez sur + Partage de fichiers pour en créer un.
    • Définissez la capacité maximale du partage.
  • Queue Storage :
    • Accédez à Files d’attente.
    • Cliquez sur + File d'attente et donnez un nom.
  • Table Storage :
    • Accédez à Tables.
    • Cliquez sur + Table et donnez un nom.

Étape 4 : Connecter une application au compte de stockage

1. Récupérer la clé d'accès

  • Dans l’onglet Clés d'accès, copiez une des clés d’accès et le nom du compte.
  • Utilisez cette clé pour autoriser les applications à accéder aux données du compte de stockage.

2. Utiliser les chaînes de connexion

  • Dans l’onglet Chaînes de connexion, copiez la chaîne complète.
  • La chaîne inclut toutes les informations nécessaires pour se connecter à votre compte.

3. Intégration avec des SDK ou outils

  3. Configurer des sauvegardes automatiques :

Étape 6 : Surveiller et gérer le compte de stockage

  2. Configurer des alertes :

Bonnes pratiques

  1. Standardisez les noms :

Le protocole SSH : Qu'est-ce que c'est ?

Le protocole SSH (Secure Shell) est un protocole réseau cryptographique permettant d'établir une connexion sécurisée entre deux machines, souvent utilisée pour administrer des systèmes à distance de manière sécurisée. SSH remplace des protocoles non sécurisés comme Telnet, FTP ou RSH, qui transmettent les données (y compris les mots de passe) en clair.

Caractéristiques principales du protocole SSH

  1. Connexion sécurisée : SSH utilise le chiffrement pour garantir la confidentialité et l'intégrité des données échangées entre le client et le serveur.

  2. Authentification forte :

    • Authentification par mot de passe.
    • Authentification par clé publique/privée (méthode recommandée).
    • Authentification multifactorielle (MFA).

  3. Fonctionnalités variées :

    • Administration à distance des serveurs via une console.
    • Transfert sécurisé de fichiers (SCP, SFTP).
    • Tunneling et redirection de ports pour accéder à des services protégés.

Comment fonctionne SSH ?

  1. Établissement de la connexion :

    • Le client SSH initie la connexion avec le serveur sur le port 22 (par défaut).
    • Le serveur SSH répond avec sa clé publique.
    • Le client vérifie cette clé publique (comparée à une liste de clés connues ou approuvées) pour s'assurer de l'authenticité du serveur.

  2. Négociation des algorithmes : Les deux parties négocient les algorithmes de chiffrement, de hachage et d'échange de clés pour sécuriser la communication.

  3. Authentification de l'utilisateur :

    • Soit avec un mot de passe.
    • Soit avec une paire de clés SSH (le client présente la clé privée associée à une clé publique préalablement enregistrée sur le serveur).

  4. Session sécurisée : Une fois la connexion établie, les données échangées sont chiffrées.

Exemple d'utilisation du protocole SSH

Connexion à un serveur distant

Vous pouvez utiliser le client SSH pour vous connecter à une machine distante. Sur un terminal, tapez :

ssh username@hostname
  • username : Nom de l'utilisateur sur la machine distante.
  • hostname : Adresse IP ou nom de domaine de la machine.

Exemple :

ssh admin@192.168.1.100

Utilisation d’une clé SSH pour se connecter

  1. Générer une paire de clés SSH :

    ssh-keygen -t rsa -b 2048

    Par défaut, cela génère deux fichiers dans ~/.ssh/ :

    • id_rsa : Clé privée (à garder secrète).
    • id_rsa.pub : Clé publique (à copier sur le serveur distant).

  2. Copier la clé publique sur le serveur :

    ssh-copy-id username@hostname

  3. Connexion sans mot de passe : Maintenant, vous pouvez vous connecter sans entrer votre mot de passe :

    ssh username@hostname

Transfert de fichiers avec SSH

  1. Utilisation de SCP (Secure Copy Protocol) :

    • Pour envoyer un fichier vers un serveur : 
      scp localfile username@hostname:/remote/directory/
    • Pour télécharger un fichier depuis un serveur : 
      scp username@hostname:/remote/file /local/directory/

  2. Utilisation de SFTP (Secure File Transfer Protocol) : Pour un accès interactif :

    sftp username@hostname

    Commandes SFTP typiques :

    • ls : Lister les fichiers sur le serveur.
    • put localfile : Envoyer un fichier vers le serveur.
    • get remotefile : Télécharger un fichier.

Fonctionnalités avancées

  1. Tunneling SSH : SSH peut rediriger un port local vers une machine distante (utile pour accéder à des services protégés).

    ssh -L local_port:target_host:target_port username@hostname

    Exemple : Accéder à une base de données MySQL sur un serveur distant :

    ssh -L 3306:localhost:3306 user@remote_server

  2. Exécution de commandes à distance : SSH permet d’exécuter directement des commandes sur un serveur sans ouvrir une session interactive.

    ssh username@hostname "commande"

    Exemple : Redémarrer un serveur :

    ssh admin@192.168.1.100 "sudo reboot"

  3. Proxy SSH : SSH peut être utilisé comme un proxy pour accéder à d'autres réseaux.

    ssh -D 8080 username@hostname

    Cela configure un proxy SOCKS sur le port 8080.

Cas d’usage du protocole SSH

  1. Administration des serveurs : Les administrateurs utilisent SSH pour gérer leurs serveurs Linux/Unix, effectuer des mises à jour ou résoudre des problèmes.

  2. Transferts sécurisés : Remplacer FTP par SCP ou SFTP pour des échanges sécurisés de fichiers.

  3. Accès sécurisé à des services internes : SSH est utilisé pour se connecter à des services hébergés dans des réseaux privés via des tunnels.

  4. Automatisation des tâches : En combinant SSH avec des outils comme Ansible ou Bash scripts, vous pouvez automatiser des déploiements ou des configurations.

Avantages du protocole SSH

  • Sécurisé : Chiffrement fort des données échangées.
  • Polyvalent : Permet la gestion distante, le transfert de fichiers, le tunneling, etc.
  • Large adoption : Prend en charge presque tous les systèmes (Windows, macOS, Linux).
  • Open Source : Implémentations comme OpenSSH sont largement utilisées et maintenues.

Le protocole RDP : Qu'est-ce que c'est ?

Le protocole RDP (Remote Desktop Protocol) est un protocole réseau développé par Microsoft pour permettre une connexion graphique à distance entre un client et une machine hôte (souvent un serveur ou un PC). Il est largement utilisé pour administrer des machines Windows, mais il peut également être utilisé pour d'autres systèmes grâce à des clients tiers.

RDP fonctionne sur le port 3389 par défaut et permet d'accéder à l'interface graphique complète d'un système distant comme si vous étiez physiquement devant celui-ci.

Principales fonctionnalités du protocole RDP

  1. Interface graphique complète : Contrairement à SSH (orienté ligne de commande), RDP permet de voir et interagir avec l'interface graphique (GUI) d'un système distant.

  2. Partage de ressources locales :

    • Accès aux disques locaux depuis la machine distante.
    • Partage des imprimantes et périphériques locaux.
    • Redirection audio pour écouter les sons générés par la machine distante.

  3. Prise en charge des multi-écrans : RDP permet de gérer plusieurs écrans connectés au client.

  4. Transferts de fichiers : Copier des fichiers entre la machine locale et distante via l'interface.

  5. Sécurité intégrée : RDP prend en charge des mécanismes comme SSL/TLS pour chiffrer les connexions, ainsi que l'authentification via NLA (Network Level Authentication).

Comment fonctionne le protocole RDP ?

  1. Établissement de la connexion :

    • Le client RDP initie une connexion vers l'hôte distant sur le port 3389.
    • L'utilisateur s'authentifie avec ses identifiants (nom d'utilisateur et mot de passe).

  2. Communication et affichage :

    • Le serveur RDP envoie des données graphiques au client.
    • Les actions du client (clavier, souris) sont envoyées au serveur.

  3. Compression et optimisation : Les données graphiques et les entrées sont compressées pour minimiser la consommation de bande passante.

Exemple d'utilisation : Connexion à distance avec RDP

Depuis un ordinateur Windows

  1. Ouvrez l'outil Connexion Bureau à distance (mstsc).
  2. Entrez l'adresse IP ou le nom de l'hôte distant (ex. : 192.168.1.100).
  3. Cliquez sur Connecter, puis entrez vos identifiants d'accès.
  4. Une fois authentifié, vous avez accès à l'interface graphique de l'hôte.

Depuis un autre système (Linux/macOS)

  • Vous pouvez utiliser des clients RDP tiers, comme :
    • Remmina (Linux).
    • Microsoft Remote Desktop (macOS et Windows).
    • FreeRDP (client open source).

Sécurité avec RDP

RDP, bien que puissant, peut être une cible pour les attaques s'il n'est pas correctement sécurisé. Voici quelques recommandations :

  1. Activer NLA (Network Level Authentication) :

    • Cela nécessite que les utilisateurs soient authentifiés avant d'établir une session RDP.
    • Réduit les risques d'attaques de force brute.

  2. Limiter l'accès via un pare-feu :

    • Autoriser uniquement des IP spécifiques à accéder au port 3389.
    • Éviter d'exposer directement RDP à Internet.

  3. Utiliser un VPN ou un Bastion :

    • Connectez-vous à un réseau privé (via VPN) pour accéder à RDP.
    • Sur Azure, utilisez un Azure Bastion, une passerelle sécurisée pour RDP et SSH sans exposer les ports publics.

    👉 Documentation Azure Bastion

  4. Changer le port RDP par défaut :

    • Modifier le port 3389 réduit les risques d'attaques automatisées.

  5. Activer l'authentification multi-facteur (MFA) :

    • Pour renforcer la sécurité des connexions.

  6. Activer des règles de verrouillage de compte :

    • Bloquer les comptes après plusieurs tentatives de connexion échouées.

Fonctionnalités avancées de RDP

  1. Sessions multiples : Sur les versions serveur de Windows (par ex., Windows Server 2019), plusieurs utilisateurs peuvent ouvrir des sessions RDP en même temps (si configuré).

  2. Accès administrateur : Les administrateurs système peuvent utiliser RDP pour résoudre des problèmes ou configurer des serveurs distants.

  3. Assistance à distance : RDP peut être utilisé pour fournir une assistance technique en temps réel.

  4. Redirection des périphériques locaux :

    • Redirigez des clés USB, imprimantes ou disques vers la machine distante pour les utiliser comme s'ils étaient connectés localement.
