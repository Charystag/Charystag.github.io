Tutoriel : Configurer un compte de stockage Azure

Les comptes de stockage Azure sont des solutions flexibles pour stocker des données dans le cloud. Ils permettent de gérer des objets (blobs), des fichiers, des disques, des tables et des files d'attente. Ce tutoriel explique comment configurer et gérer un compte de stockage Azure.

Étape 1 : Comprendre les types de stockage Azure

Avant de commencer, comprenez les principaux types de stockage offerts par Azure :

  1. Blob Storage : Stockage d’objets pour les données non structurées (images, vidéos, sauvegardes).
  2. File Storage : Partages de fichiers accessibles via SMB ou NFS.
  3. Queue Storage : Files d'attente de messages pour la communication asynchrone.
  4. Table Storage : Stockage NoSQL pour les données structurées.

Étape 2 : Créer un compte de stockage Azure

1. Accéder au portail Azure

  • Connectez-vous au portail Azure.
  • Recherchez Comptes de stockage dans la barre de recherche et cliquez sur l’option correspondante.

2. Créer un compte de stockage

  • Cliquez sur + Créer.
  • Remplissez les informations nécessaires :

Onglet "De base"

  • Abonnement : Sélectionnez l'abonnement Azure à utiliser.
  • Groupe de ressources : Choisissez un groupe existant ou créez-en un nouveau.
  • Nom du compte de stockage : Donnez un nom unique (entre 3 et 24 caractères, uniquement alphanumériques).
  • Région : Sélectionnez la région Azure où vous souhaitez héberger le compte.
  • Performance :
    • Standard : Idéal pour des applications courantes avec des besoins modérés en performance.
    • Premium : Pour des applications nécessitant des performances élevées (par exemple, bases de données).
  • Redondance :
    • LRS (Locally Redundant Storage) : Redondance dans une seule région.
    • GRS (Geo-Redundant Storage) : Redondance géographique pour la haute disponibilité.

Onglets supplémentaires (facultatif)

  • Réseau : Configurez l’accès privé ou public.

  • Sécurité : Activez le chiffrement avec des clés gérées par le client ou Azure.

  • Avancé : Configurez des fonctionnalités telles que le stockage hiérarchisé.

  • Une fois tous les champs remplis, cliquez sur Vérifier + Créer puis sur Créer.

Étape 3 : Configurer le compte de stockage

1. Accéder au compte de stockage

  • Une fois créé, cliquez sur le compte dans la liste des Comptes de stockage.

2. Configurer des services spécifiques

  • Blob Storage :
    • Allez dans l'onglet Conteneurs.
    • Cliquez sur + Conteneur pour créer un nouveau conteneur.
    • Définissez les permissions d'accès (Public/Privé).
  • File Storage :
    • Accédez à Partages de fichiers.
    • Cliquez sur + Partage de fichiers pour en créer un.
    • Définissez la capacité maximale du partage.
  • Queue Storage :
    • Accédez à Files d’attente.
    • Cliquez sur + File d'attente et donnez un nom.
  • Table Storage :
    • Accédez à Tables.
    • Cliquez sur + Table et donnez un nom.

Étape 4 : Connecter une application au compte de stockage

1. Récupérer la clé d'accès

  • Dans l’onglet Clés d'accès, copiez une des clés d’accès et le nom du compte.
  • Utilisez cette clé pour autoriser les applications à accéder aux données du compte de stockage.

2. Utiliser les chaînes de connexion

  • Dans l’onglet Chaînes de connexion, copiez la chaîne complète.
  • La chaîne inclut toutes les informations nécessaires pour se connecter à votre compte.

3. Intégration avec des SDK ou outils

  • Azure Storage peut être utilisé avec les SDKs (disponibles pour .NET, Python, Java, etc.).
  • Exemple pour Python : 
    from azure.storage.blob import BlobServiceClient

connection_string = "votre_chaine_de_connexion"
blob_service_client = BlobServiceClient.from_connection_string(connection_string)

container_client = blob_service_client.get_container_client("nom_du_conteneur")

Étape 5 : Configurer des règles de sécurité et des stratégies

  1. Activer les pare-feu et les réseaux virtuels :

    • Dans l'onglet Mise en réseau, autorisez uniquement des adresses IP spécifiques ou des sous-réseaux Azure à accéder au compte.

  2. Activer le versioning des blobs :

    • Dans l'onglet Configuration, activez le Versionnement des blobs pour conserver les versions précédentes des objets.

  3. Configurer des sauvegardes automatiques :

    • Utilisez Azure Backup pour planifier des sauvegardes régulières des données dans le compte de stockage.

Étape 6 : Surveiller et gérer le compte de stockage

  1. Analyse des métriques :
    • Accédez à l’onglet Métriques pour surveiller les performances (latence, débit, etc.).
  2. Configurer des alertes :
    • Allez dans Alertes pour recevoir des notifications en cas de dépassement des seuils (par exemple, utilisation du stockage).

Bonnes pratiques

  1. Standardisez les noms :
    • Utilisez des conventions claires pour nommer les conteneurs et partages.
  2. Sécurisez les données :
    • Activez l’accès privé ou limitez l'accès par réseau.
  3. Optimisez les coûts :
    • Configurez des règles de cycle de vie pour supprimer automatiquement les données obsolètes.
  4. Testez la récupération :
    • Vérifiez régulièrement que vous pouvez accéder et récupérer les données sauvegardées.
