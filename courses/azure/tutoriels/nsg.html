Panorama du Cloud - Azure

Filtrer le trafic réseau avec un groupe de sécurité réseau (NSG)

Un Network Security Group (NSG) dans Azure est un composant essentiel pour sécuriser le trafic réseau. Il permet de contrôler l’accès aux ressources Azure en filtrant les flux entrants et sortants sur la base de règles configurables. Cela vous aide à renforcer la sécurité de vos ressources tout en respectant les exigences réseau de vos applications.

Vue d'ensemble

Un NSG agit comme un pare-feu virtuel qui vous permet de définir des règles pour autoriser ou bloquer le trafic réseau vers des sous-réseaux ou des interfaces réseau spécifiques dans Azure. Cela inclut des scénarios tels que :

  • Restreindre l'accès à une machine virtuelle (VM) à partir de plages d'IP spécifiques.
  • Limiter les communications inter-sous-réseaux dans un réseau virtuel.
  • Bloquer des ports spécifiques pour renforcer la sécurité.

Scénarios courants

Limiter le trafic entrant

  • Par exemple, n’autoriser que des adresses IP spécifiques à accéder à une application web sur le port 80 (HTTP) ou 443 (HTTPS).

Contrôler le trafic sortant

  • Empêcher une VM de se connecter à des destinations externes non autorisées, comme des domaines malveillants.

Appliquer des règles granulaires

  • Créer des règles basées sur des plages d’adresses IP, des ports ou des protocoles (TCP/UDP).

Priorisation des règles

  • Les règles NSG sont évaluées par priorité, où les valeurs les plus basses sont traitées en premier.

Conseils pour réussir

Planifiez vos règles avec soin

  • Déterminez à l'avance quels flux de trafic sont nécessaires pour vos ressources.
  • Utilisez les groupes de sécurité par défaut et minimisez les règles globales trop permissives.

Utilisez des balises Azure (tags)

  • Azure offre des balises intégrées comme Internet, VirtualNetwork, ou AzureLoadBalancer, qui simplifient la configuration des règles.

Surveillez l’impact des règles

  • Testez les modifications avec précaution pour éviter des interruptions de service. Utilisez les journaux NSG et Azure Monitor pour vérifier l’efficacité des règles.

Combinez NSG et d'autres outils de sécurité

  • Un NSG est efficace pour le trafic réseau au niveau de la couche 3 (IP) et 4 (TCP/UDP), mais vous pouvez renforcer la sécurité avec des outils comme Azure Firewall pour un contrôle plus avancé.

Évitez les règles conflictuelles

  • Si plusieurs règles sont appliquées à un NSG, assurez-vous qu’elles ne se contredisent pas, en particulier pour des ports critiques comme 22 (SSH) ou 3389 (RDP).
