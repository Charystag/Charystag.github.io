Tutoriel : Utiliser Azure Blob Storage pour stocker et gérer des fichiers

Azure Blob Storage est un service de stockage d’objets conçu pour stocker des données non structurées, comme des documents, des images, des vidéos, et des sauvegardes. Ce guide vous montre comment configurer et utiliser Azure Blob Storage pour stocker et gérer vos fichiers.

Étape 1 : Comprendre les types de blobs

Azure Blob Storage propose trois types de blobs pour différents cas d’utilisation :

  1. Blobs de bloc (Block Blob) : Idéal pour stocker des fichiers (documents, médias, etc.).
  2. Blobs d’ajout (Append Blob) : Utilisé pour les fichiers devant recevoir des ajouts fréquents (par exemple, fichiers journaux).
  3. Blobs de pages (Page Blob) : Optimisé pour les mises à jour fréquentes (comme les disques de machines virtuelles).

Étape 2 : Créer un compte de stockage Azure

  1. Connectez-vous au portail Azure : https://portal.azure.com.
  2. Recherchez "Comptes de stockage" et cliquez sur Créer un compte de stockage.
  3. Configurez le compte de stockage :
    • Performance : Standard ou Premium.
    • Redondance : LRS, ZRS, GRS, ou RA-GRS selon vos besoins.
  4. Une fois les options sélectionnées, cliquez sur Créer.

Étape 3 : Configurer un conteneur pour les blobs

  1. Accédez au compte de stockage créé.
  2. Dans le menu latéral, sélectionnez Conteneurs.
  3. Cliquez sur + Conteneur pour créer un nouveau conteneur :
    • Nom : Donnez un nom explicite (ex. : documents, images).
    • Niveau d'accès public :
      • Privé : Accessible uniquement via des autorisations spécifiques.
      • Public (lecture) : Tous les utilisateurs peuvent lire les blobs.
  4. Cliquez sur Créer.

Étape 4 : Télécharger des fichiers dans Azure Blob Storage

Depuis le portail Azure

  1. Accédez à votre conteneur.
  2. Cliquez sur Télécharger et sélectionnez un fichier à télécharger depuis votre ordinateur.
  3. Une fois téléchargé, le fichier apparaîtra dans la liste des blobs du conteneur.

Depuis Azure CLI

  1. Ouvrez une fenêtre de terminal.
  2. Connectez-vous à Azure : 
    az login
  3. Téléchargez un fichier vers un conteneur : 
    az storage blob upload --account-name <NomDuCompte> --container-name <NomDuConteneur> --name <NomDuFichier> --file <CheminDuFichier>
    Exemple : 
    az storage blob upload --account-name moncompte --container-name documents --name rapport.pdf --file /home/user/rapport.pdf

Depuis un script Python

  1. Installez le SDK Azure pour Python : 
    pip install azure-storage-blob
  2. Téléchargez un fichier : 
    from azure.storage.blob import BlobServiceClient

connection_string = "votre_chaine_de_connexion"
blob_service_client = BlobServiceClient.from_connection_string(connection_string)
container_client = blob_service_client.get_container_client("nom_du_conteneur")

with open("chemin_du_fichier", "rb") as fichier:
    container_client.upload_blob(name="nom_du_blob", data=fichier)

Étape 5 : Télécharger ou gérer les fichiers stockés

Télécharger un fichier

  • Dans le portail Azure :

    • Sélectionnez le fichier dans le conteneur.
    • Cliquez sur Télécharger pour le récupérer.

  • Avec Azure CLI :

    az storage blob download --account-name <NomDuCompte> --container-name <NomDuConteneur> --name <NomDuBlob> --file <CheminLocal>

    Exemple :

    az storage blob download --account-name moncompte --container-name documents --name rapport.pdf --file ./rapport_local.pdf

Supprimer un fichier

  • Dans le portail Azure :
    • Sélectionnez le blob, cliquez sur Supprimer.
  • Avec Azure CLI : 
    az storage blob delete --account-name <NomDuCompte> --container-name <NomDuConteneur> --name <NomDuBlob>

Étape 6 : Configurer des règles de cycle de vie

  1. Accédez à l’onglet Cycle de vie dans le compte de stockage.
  2. Configurez des règles pour automatiser la gestion des fichiers :
    • Déplacement vers le stockage d’archives après X jours.
    • Suppression automatique des fichiers après une certaine durée.

Étape 7 : Contrôler l’accès aux blobs

  1. Utiliser des clés d'accès :

    • Accédez à l’onglet Clés d'accès pour récupérer une clé permettant d'accéder au compte.
    • Utilisez cette clé dans vos applications pour interagir avec les blobs.

  2. Générer un lien SAS (Shared Access Signature) :

    • Accédez à l’onglet Signature d'accès partagé (SAS).
    • Configurez les permissions (lecture, écriture, suppression).
    • Générez un lien SAS pour partager temporairement l'accès à un fichier.

Étape 8 : Surveiller et analyser l’utilisation

  1. Accédez à l’onglet Métriques dans le compte de stockage pour consulter les statistiques (latence, utilisation, débit).
  2. Configurez des alertes dans Alertes pour être notifié en cas d'anomalies ou de dépassement des quotas.

Bonnes pratiques pour Azure Blob Storage

  1. Sécurisez vos données :
    • Utilisez des conteneurs privés et activez le chiffrement.
    • Générez des liens SAS pour partager l'accès de manière temporaire.
  2. Automatisez la gestion des fichiers :
    • Configurez des règles de cycle de vie pour réduire les coûts.
  3. Utilisez des conventions de nommage claires :
    • Organisez les blobs avec des noms significatifs et hiérarchiques (ex. : 2024/images/projetA/image1.jpg).
  4. Surveillez vos coûts :
    • Suivez l'utilisation du stockage dans Analyse des coûts.
