Tutoriel : Déployer une machine virtuelle Windows/Linux sur Azure

Les machines virtuelles (VM) dans Azure permettent de créer et de gérer des serveurs virtuels exécutant Windows ou Linux. Ce tutoriel vous guidera à travers les étapes de déploiement d'une VM depuis le portail Azure.

Étape 1 : Accéder au portail Azure

  1. Connectez-vous à votre compte sur le portail Azure.
  2. Dans la barre de recherche, tapez "Machines virtuelles" et cliquez sur le service correspondant.

Étape 2 : Créer une nouvelle machine virtuelle

  1. Cliquer sur "Créer" :

    • Dans la section Machines virtuelles, cliquez sur + Créer puis Machine virtuelle Azure.

  2. Configuration de base : Remplissez les champs suivants :

    • Abonnement : Sélectionnez l’abonnement Azure à utiliser.
    • Groupe de ressources : Choisissez un groupe existant ou créez-en un nouveau.
    • Nom de la machine virtuelle : Entrez un nom explicite pour la VM.
    • Région : Sélectionnez la région Azure où vous souhaitez héberger la VM.
    • Options de disponibilité : Configurez la redondance pour garantir la haute disponibilité (facultatif).

  3. Système d’exploitation :

    • Sélectionnez le système d’exploitation :
      • Windows Server : Ex. : Windows Server 2019.
      • Linux : Ex. : Ubuntu Server 20.04 LTS.

  4. Taille de la machine virtuelle :

    • Cliquez sur Changer de taille et sélectionnez une taille en fonction des besoins (CPU, RAM).
    • Exemples courants :
      • B1ms pour des tests de base.
      • D2s_v3 pour des charges de travail standard.

  5. Options d'authentification :

    • Pour Windows : Sélectionnez l’authentification par nom d’utilisateur et mot de passe ou une clé SSH (facultatif).
    • Pour Linux : Utilisez une clé SSH (recommandée) ou un mot de passe.
      • Si vous choisissez une clé SSH, générez-en une via Azure ou utilisez votre propre clé.

Étape 3 : Configuration des disques

  1. Disque du système d’exploitation :

    • Sélectionnez un type de disque : SSD Premium, SSD Standard ou HDD Standard.
    • SSD Premium est recommandé pour des performances élevées.

  2. Disques de données (facultatif) :

    • Cliquez sur Créer et attacher un nouveau disque si vous avez besoin de stockage supplémentaire.

Étape 4 : Configuration réseau

  1. Réseau virtuel :

    • Azure configure automatiquement un réseau virtuel par défaut. Vous pouvez également créer un réseau personnalisé.

  2. Sous-réseau :

    • Vérifiez que le sous-réseau par défaut est sélectionné.

  3. Adresse IP publique :

    • Activez l'adresse IP publique pour permettre l'accès à distance.

  4. Port d’entrée :

    • Pour Windows, ouvrez le port RDP (3389) pour l'accès à distance.
    • Pour Linux, ouvrez le port SSH (22).

Étape 5 : Vérification et déploiement

  1. Options avancées (facultatif) :

    • Configurez des extensions, des scripts de démarrage ou d'autres paramètres avancés selon vos besoins.

  2. Revue et création :

    • Cliquez sur Vérifier + créer.
    • Azure validera vos configurations.
    • Cliquez sur Créer pour démarrer le déploiement.

Étape 6 : Connexion à la machine virtuelle

  1. Pour Windows :

    • Accédez à la VM dans le portail Azure.
    • Cliquez sur Se connecter > RDP.
    • Téléchargez le fichier RDP, ouvrez-le avec votre client RDP, et connectez-vous avec vos identifiants.

  2. Pour Linux :

    • Accédez à la VM dans le portail Azure.
    • Cliquez sur Se connecter > SSH.
    • Suivez les instructions pour utiliser la clé SSH et la commande appropriée : 
      ssh <NomUtilisateur>@<AdresseIP>

Étape 7 : Gérer la machine virtuelle

  1. Arrêter/Démarrer :

    • Vous pouvez arrêter ou redémarrer la VM depuis le portail.

  2. Superviser les performances :

    • Allez dans l’onglet Surveillance pour consulter les métriques (CPU, RAM, etc.).

  3. Sauvegardes :

    • Configurez des sauvegardes dans l’onglet Sauvegarde pour éviter la perte de données.

  4. Sécurité :

    • Vérifiez régulièrement les règles de pare-feu.
    • Utilisez Azure Bastion pour une connexion plus sécurisée.

Bonnes pratiques

  1. Fermer les ports inutilisés :

    • Ne laissez ouverts que les ports nécessaires (3389 pour RDP, 22 pour SSH).

  2. Automatisez :

    • Utilisez des outils comme Azure CLI, PowerShell ou des modèles ARM pour déployer des VMs plus rapidement.

  3. Gérez les coûts :

    • Arrêtez les VMs inutilisées pour réduire les coûts.
    • Surveillez les dépenses avec Azure Cost Management.

  4. Maintenez vos VMs à jour :

    • Appliquez les mises à jour système pour améliorer la sécurité et les performances.
