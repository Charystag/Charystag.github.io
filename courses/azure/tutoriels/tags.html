Tutoriel : Déploiement et gestion des étiquettes (tags) pour une meilleure organisation dans Azure

Les étiquettes (tags) dans Microsoft Azure sont des paires clé-valeur que vous pouvez appliquer à des ressources, des groupes de ressources, et des abonnements pour améliorer leur organisation et leur gestion. Ce tutoriel vous guide à travers le déploiement et la gestion des étiquettes dans Azure.

Pourquoi utiliser les étiquettes dans Azure ?

  1. Organisation : Classez et identifiez les ressources par projet, équipe, environnement (prod/dev/test), ou toute autre catégorie.
  2. Analyse des coûts : Attribuez des coûts à des projets ou départements spécifiques.
  3. Automatisation : Simplifiez les opérations avec des scripts et des stratégies basés sur les étiquettes.
  4. Gestion centralisée : Filtrez et recherchez rapidement des ressources dans le portail Azure.

Étape 1 : Comprendre les étiquettes

Une étiquette se compose de deux parties :

  • Clé : Le nom ou la catégorie de la étiquette (ex. : Projet, Environnement, Equipe).
  • Valeur : L’information associée à la clé (ex. : SiteWeb, Production, Marketing).

Exemple :
Clé : Projet → Valeur : SiteWeb

Étape 2 : Ajouter des étiquettes à une ressource

  1. Depuis le Portail Azure :

    • Accédez à une ressource (par exemple, une machine virtuelle ou un compte de stockage).
    • Cliquez sur Étiquettes dans le menu latéral.
    • Ajoutez des paires clé-valeur :
      • Exemple : Clé : Environnement → Valeur : Production
    • Cliquez sur Enregistrer.

  2. Lors de la création d'une ressource :

    • Lors de la configuration d’une nouvelle ressource, cherchez l’onglet ou la section Étiquettes.
    • Ajoutez les étiquettes nécessaires avant de finaliser la création.

Étape 3 : Ajouter des étiquettes à plusieurs ressources simultanément

  1. Utilisation du portail Azure :

    • Dans la barre de recherche du portail, tapez Étiquettes et sélectionnez l'option.
    • Cliquez sur Ajouter une étiquette à des ressources.
    • Sélectionnez plusieurs ressources dans la liste.
    • Ajoutez les étiquettes souhaitées (Clé/valeur).
    • Cliquez sur Appliquer.

  2. Avec Azure PowerShell : Utilisez PowerShell pour automatiser l’application de étiquettes :

    Set-AzResource -ResourceId "/subscriptions/<SubscriptionID>/resourceGroups/<ResourceGroupName>/providers/<ResourceType>/<ResourceName>" `
-Tag @{ "Projet" = "SiteWeb"; "Environnement" = "Production" }

  3. Avec Azure CLI : Vous pouvez également utiliser Azure CLI :

    az resource tag --ids "/subscriptions/<SubscriptionID>/resourceGroups/<ResourceGroupName>/providers/<ResourceType>/<ResourceName>" \
--tags Projet=SiteWeb Environnement=Production

Étape 4 : Gérer et modifier les étiquettes

  1. Modifier des étiquettes :

    • Accédez à une ressource ou un groupe de ressources.
    • Cliquez sur Étiquettes, modifiez les paires clé-valeur existantes ou ajoutez-en de nouvelles.
    • Enregistrez vos modifications.

  2. Supprimer des étiquettes :

    • Supprimez une clé ou une valeur dans la section Étiquettes.
    • Cliquez sur Enregistrer pour confirmer.

Étape 5 : Utiliser Azure Policy pour appliquer des étiquettes automatiquement

Azure Policy permet de définir des règles pour garantir que toutes les ressources ont les étiquettes nécessaires. Par exemple, vous pouvez exiger que toutes les ressources dans un groupe de ressources aient une étiquette Environnement.

  1. Créer une stratégie pour les étiquettes :

    • Accédez à Azure Policy dans le portail.
    • Cliquez sur Définitions > + Ajouter une définition personnalisée.
    • Utilisez une définition intégrée ou personnalisez une règle. Exemple d'une définition intégrée : Ajouter une étiquette à une ressource lors de son création.
    • Spécifiez les détails de la stratégie (clé/valeur).

  2. Affecter la stratégie :

    • Allez dans Assignations dans Azure Policy.
    • Affectez la stratégie à un abonnement ou un groupe de ressources.

  3. Automatiser les corrections :

    • Configurez la stratégie pour appliquer automatiquement les étiquettes manquantes.

Étape 6 : Surveiller les ressources par étiquettes

  1. Filtrer les ressources par étiquettes :

    • Accédez à la section Tous les services > Ressources.
    • Cliquez sur Filtrer et choisissez une étiquette (ex. : Projet=SiteWeb).

  2. Analyse des coûts par étiquettes :

    • Accédez à Analyse des coûts.
    • Configurez un rapport de coût basé sur les étiquettes (ex. : afficher les coûts par environnement).

Bonnes pratiques pour les étiquettes

  1. Standardisez les clés : Utilisez des noms cohérents et explicites (Projet, Environnement, Equipe) pour éviter les doublons.
  2. Automatisez l’application : Utilisez Azure Policy pour éviter les oublis.
  3. Limitez les clés personnalisées : Utilisez des clés communes pour simplifier la gestion.
  4. Surveillez et nettoyez régulièrement : Vérifiez les ressources avec des étiquettes incomplètes ou incorrectes.
