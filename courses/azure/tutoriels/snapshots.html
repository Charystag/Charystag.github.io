Tutoriel : Gérer les sauvegardes et les snapshots des Machines Virtuelles (VMs) sur Azure

La gestion des sauvegardes et des snapshots des machines virtuelles (VMs) dans Azure est essentielle pour garantir la continuité des opérations et éviter les pertes de données en cas de panne. Ce tutoriel explique comment configurer et gérer les sauvegardes et les snapshots des VMs.

1. Différence entre sauvegardes et snapshots

Sauvegardes

  • Description : Les sauvegardes Azure (via le service Azure Backup) permettent de créer des copies cohérentes de vos VMs, incluant le système d'exploitation, les disques, et les données.
  • Utilisation : Restauration complète ou partielle en cas de panne, de suppression accidentelle ou de corruption de données.
  • Avantages :
    • Gestion centralisée.
    • Protection automatisée avec des stratégies de rétention.
    • Chiffrement des sauvegardes.

Snapshots

  • Description : Un snapshot est une copie ponctuelle d’un disque VM spécifique (généralement un disque OS ou un disque de données).
  • Utilisation : Revenir rapidement à un état précédent, cloner des disques, ou tester des modifications.
  • Avantages :
    • Plus rapide à créer que les sauvegardes.
    • Utile pour des sauvegardes temporaires ou des tests.

2. Gérer les sauvegardes d'une VM avec Azure Backup

Étape 1 : Configurer une sauvegarde pour une VM

  1. Accéder à la VM :

    • Dans le portail Azure, recherchez et sélectionnez votre machine virtuelle.

  2. Configurer Azure Backup :

    • Dans le menu latéral, cliquez sur Sauvegarde.
    • Sélectionnez un Coffre Recovery Services existant ou créez-en un :
      • Donnez un nom au coffre.
      • Sélectionnez une région et un abonnement.

  3. Configurer la stratégie de sauvegarde :

    • Azure propose une stratégie par défaut :
      • Sauvegarde quotidienne.
      • Conservation des sauvegardes pendant 30 jours.
    • Vous pouvez personnaliser les paramètres :
      • Fréquence (quotidienne/hebdomadaire).
      • Durée de conservation (jours, semaines, mois, années).

  4. Activer la sauvegarde :

    • Cliquez sur Activer la sauvegarde pour démarrer la protection de la VM.

Étape 2 : Restauration depuis une sauvegarde

  1. Accéder au Coffre Recovery Services :

    • Allez dans Coffres Recovery Services depuis le portail Azure.
    • Sélectionnez le coffre contenant la sauvegarde.

  2. Choisir l'élément à restaurer :

    • Cliquez sur Articles sauvegardés.
    • Sélectionnez la VM et choisissez un point de restauration.

  3. Options de restauration :

    • Créer une nouvelle VM : Crée une nouvelle VM à partir de la sauvegarde.
    • Restauration des disques : Restaure uniquement les disques, que vous pouvez attacher à une VM existante.

3. Gérer les snapshots des disques VM

Étape 1 : Créer un snapshot

  1. Accéder au disque de la VM :

    • Dans le portail Azure, ouvrez la VM.
    • Cliquez sur Disques dans le menu.
    • Sélectionnez le disque que vous voulez snapshotter (ex. : disque OS ou disque de données).

  2. Créer un snapshot :

    • Dans l'onglet Vue d'ensemble du disque, cliquez sur Créer un snapshot.
    • Configurez les paramètres :
      • Nom : Donnez un nom explicite (ex. : Snapshot-Disque-Prod).
      • Type de snapshot : Sélectionnez Snapshot complet (copie complète) ou Snapshot incrémental (plus rapide et moins coûteux).
      • Compte de stockage : Spécifiez où stocker le snapshot.

  3. Créer :

    • Cliquez sur Créer pour démarrer la création du snapshot.

Étape 2 : Utiliser un snapshot

  1. Créer un nouveau disque depuis un snapshot :

    • Allez dans le service Snapshots dans le portail Azure.
    • Sélectionnez le snapshot souhaité.
    • Cliquez sur Créer un disque et configurez les options (type, taille, etc.).
    • Le disque créé peut ensuite être attaché à une VM.

  2. Remplacer un disque existant :

    • Arrêtez la VM.
    • Détachez le disque existant.
    • Attachez le disque restauré à partir du snapshot.

Étape 3 : Supprimer un snapshot

  1. Accéder au service Snapshots :

    • Recherchez Snapshots dans le portail Azure.
    • Sélectionnez le snapshot que vous souhaitez supprimer.

  2. Supprimer :

    • Cliquez sur Supprimer pour libérer de l'espace de stockage.

4. Bonnes pratiques pour les sauvegardes et snapshots

Sauvegardes

  1. Automatisez les sauvegardes :
    • Utilisez des stratégies pour garantir que toutes les VMs critiques sont protégées.
  2. Surveillez l'état des sauvegardes :
    • Configurez des alertes pour les échecs de sauvegarde.
  3. Testez régulièrement les restaurations :
    • Assurez-vous que les points de restauration sont valides et utilisables.

Snapshots

  1. Utilisez des snapshots pour des modifications rapides :
    • Avant une mise à jour majeure ou des tests.
  2. Nettoyez régulièrement :
    • Supprimez les snapshots obsolètes pour économiser les coûts de stockage.
  3. Combinez snapshots et sauvegardes :
    • Utilisez les sauvegardes pour des restaurations complètes et les snapshots pour des interventions rapides.
