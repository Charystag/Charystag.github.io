Tutoriel : Connexion à une Machine Virtuelle (VM) avec RDP ou SSH sur Azure

Se connecter à une machine virtuelle sur Azure permet d’accéder à son interface pour la configurer, exécuter des applications, ou effectuer des maintenances. Ce guide vous explique comment se connecter à une VM, que ce soit avec Remote Desktop Protocol (RDP) pour Windows ou Secure Shell (SSH) pour Linux.

Prérequis pour la connexion

  1. Pour RDP (Windows) :

    • Une VM Windows déployée.
    • Le port 3389 doit être ouvert dans les paramètres réseau de la VM.
    • Vous devez disposer des identifiants de connexion (nom d'utilisateur et mot de passe).

  2. Pour SSH (Linux) :

    • Une VM Linux déployée.
    • Le port 22 doit être ouvert dans les paramètres réseau de la VM.
    • Vous devez disposer des clés SSH ou des identifiants utilisateur/mot de passe.

  3. Adresse IP publique :

    • La VM doit avoir une adresse IP publique pour une connexion externe.

Étape 1 : Vérifier les configurations réseau de la VM

Avant de vous connecter :

  1. Accédez à votre VM dans le portail Azure.
  2. Allez dans l’onglet Réseau :
    • Vérifiez que l'adresse IP publique est activée.
    • Confirmez que les ports nécessaires (3389 pour RDP ou 22 pour SSH) sont ouverts dans la groupe de sécurité réseau (NSG).

Étape 2 : Connexion à une VM Windows via RDP

1. Récupérer l'adresse IP

  • Accédez à votre VM sur le portail Azure.
  • Dans l'onglet Vue d’ensemble, notez l’adresse IP publique.

2. Télécharger le fichier RDP

  • Dans le portail Azure, cliquez sur Se connecter > RDP.
  • Configurez les paramètres si nécessaire (port, adresse IP, etc.).
  • Téléchargez le fichier .rdp.

3. Ouvrir la connexion RDP

  • Double-cliquez sur le fichier .rdp téléchargé.
  • Cliquez sur Se connecter si une fenêtre d’avertissement s’affiche.

4. Saisir les identifiants

  • Entrez le nom d'utilisateur et le mot de passe configurés lors de la création de la VM.
  • Cliquez sur OK.

5. Accès à la VM

  • Vous serez connecté à votre VM et pourrez accéder à son interface comme si vous étiez physiquement devant le serveur.

Étape 3 : Connexion à une VM Linux via SSH

1. Récupérer l'adresse IP

  • Accédez à votre VM Linux dans le portail Azure.
  • Notez l'adresse IP publique dans l'onglet Vue d’ensemble.

2. Utiliser une clé SSH (recommandé)

  1. Avec un terminal sous Windows/Linux/Mac :

    • Ouvrez un terminal.
    • Tapez la commande suivante : 
      ssh <NomUtilisateur>@<AdresseIP>
      Exemple : 
      ssh azureuser@20.80.50.100

  2. Si une clé privée est nécessaire :

    • Ajoutez le chemin vers la clé privée dans la commande : 
      ssh -i /chemin/de/votre/cle_privee.pem azureuser@<AdresseIP>

  3. Accepter l'empreinte numérique :

    • Lors de la première connexion, vous recevrez une demande pour accepter l'empreinte de la machine. Tapez yes.

3. Connexion avec un mot de passe (optionnel) :

  • Si vous n’utilisez pas de clé SSH, tapez simplement la commande suivante et fournissez le mot de passe : 
    ssh <NomUtilisateur>@<AdresseIP>

Étape 4 : Dépannage de connexion

Pour RDP :

  • Vérifiez que le port 3389 est ouvert :

    • Dans la section Réseau, assurez-vous qu'une règle autorise Port 3389 dans les Groupe de sécurité réseau (NSG).

  • Adresse IP publique :

    • Assurez-vous que votre VM a une adresse IP publique ou utilisez Azure Bastion pour une connexion sécurisée sans exposer l’IP publique.

  • Pare-feu Windows :

    • Confirmez que RDP est autorisé dans le pare-feu Windows de la VM.

Pour SSH :

  • Port 22 :

    • Vérifiez que le port 22 est ouvert dans les règles NSG.

  • Permission de la clé SSH :

    • La clé privée utilisée pour la connexion SSH doit avoir les bonnes permissions : 
      chmod 600 /chemin/de/votre/cle_privee.pem

  • Utilisateur valide :

    • Confirmez que vous utilisez le bon nom d’utilisateur (exemple : azureuser par défaut sur Ubuntu).

Étape 5 : Bonnes pratiques pour la connexion

  1. Sécurisez vos connexions :

    • Utilisez une clé SSH plutôt qu’un mot de passe pour les VMs Linux.
    • Configurez Azure Bastion pour éviter d'exposer vos ports RDP/SSH au public.

  2. Restreignez l'accès :

    • Modifiez les règles du NSG pour n’autoriser que votre adresse IP à se connecter à la VM.

  3. Surveillez les connexions :

    • Utilisez Azure Monitor pour superviser les tentatives de connexion et détecter les anomalies.
