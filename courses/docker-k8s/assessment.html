Assessment: Dockerize a Simple Web Application (Manual & Compose)

All your Dockerfiles must use the antepenultimate version of your base image

Overall Goal:

Understand and implement the deployment of a 3-container web application architecture (Nginx Reverse Proxy, Web Application, MySQL Database) using both individual Docker commands and Docker Compose, meeting the specified requirements.

Common Requirements (Apply to both Part 1 and Part 2):

The target architecture must adhere to the following specifications:

  1. Web Application:

    • A simple web application (language of choice) must be developed.
    • The application must implement at least two routes:
      • /health: returns 200 OK with a JSON response (e.g., {"status": "healthy"}).
      • /data: fetches and displays test data from the MySQL database.
    • The application server listens only on internal port 4743.

  2. Target Container Architecture:

    • Nginx Container:
      • Based on the official nginx image. Configuration may require a custom Dockerfile or volume mount.
      • Acts as a reverse proxy.
      • Maps host port 5423 to container port 824.
      • Forwards requests for /health and /data to the application container (app service on port 4743).
      • Connected to site_network.
    • Application Container:
      • Built from the app/Dockerfile. The target image name is app.
      • The app/Dockerfile must use the official alpine image as its parent image.
      • Connects to the mysql service via db_network.
      • Connected to db_network and site_network.
    • MySQL Container:
      • Built from the mysql/Dockerfile. The target image name is mysql.
      • Configured to listen internally on port 5655.
      • Data persisted using the db_volume volume.
      • Connected to db_network.
      • Must include a functional healthcheck defined within its Dockerfile.

  3. Volumes and Networks:

    • A volume named db_volume must be used for MySQL data persistence.
    • Two Docker networks must be created: db_network and site_network.

  4. Naming Conventions:

    • Image names must be: app, mysql. (Use official nginx unless custom build needed for config, then nginx).
    • Container names and network aliases must be configured as follows:
      • MySQL container: name mysql_container, alias mysql on db_network.
      • App container: alias app on site_network (container name can be default or specified).
      • Nginx container: name nginx_container.

  5. Directory Structure:

    • The project must be organized as follows:

Part 1: Manual Docker Commands

Objective:

Create and run the specified architecture using individual docker commands, demonstrating the manual setup process.

Instructions:

  1. Create a script file (setup_manual.sh for Bash or setup_manual.ps1 for PowerShell).
  2. The script must contain the sequence of docker commands necessary to:
    • Create the db_network and site_network networks.
    • Create the db_volume volume.
    • Build the mysql image from mysql/Dockerfile.
    • Build the app image from app/Dockerfile (which starts FROM alpine:...).
    • Build the nginx image only if nginx/Dockerfile is used for configuration.
    • Run the mysql container (docker run ...) meeting all requirements specified in "Common Requirements" (name, network, alias, volume, internal port, env vars). Run detached.
    • Run the app container (docker run ...) meeting all requirements (networks, connectivity to mysql, internal port, env vars). Run detached.
    • Run the nginx container (docker run ...) meeting all requirements (network, port mapping, config mount if needed, proxying to app). Run detached.
  3. The script must include commented-out commands (or a separate cleanup script) to stop/remove the containers, remove the networks, and remove the volume created.

Deliverable for Part 1:

  • The script file (setup_manual.sh or setup_manual.ps1).

Part 2: Docker Compose

Objective:

Define and run the identical architecture declaratively using a Docker Compose file (compose.yaml), including service dependencies and healthchecks.

Instructions:

  1. Create the compose.yaml file in the project's root directory.
  2. Define the three services (mysql, app, nginx).
  3. Define the top-level networks (db_network, site_network) and volumes (db_volume).
  4. Configure each service according to the "Common Requirements" and the following specifics:
    • mysql service: Specify build context, image, container_name, networks, volumes, environment variables, and reference the healthcheck from its Dockerfile.
    • app service: Specify build context, image, networks, environment variables. Implement depends_on referencing the mysql service with condition: service_healthy. Define an appropriate healthcheck within the service definition here (testing the /health endpoint). Remember its Dockerfile starts FROM alpine.
    • nginx service: Specify the official nginx image (or build context if needed), container_name, ports mapping, networks, volume mount for configuration. Implement depends_on referencing the app service with condition: service_healthy.
  5. The application must run correctly when started with docker compose up --build -d.

Deliverable for Part 2:

  • The functioning compose.yaml file.

Submission Instructions:

  • Please submit your entire project (including setup_manual.(sh|ps1), compose.yaml, all Dockerfiles, application source code src/, and any configuration files conf/) as a public GitHub repository.
  • Ensure your repository's README.md file briefly explains how to run both Part 1 (the script) and Part 2 (Docker Compose).
  • Provide the link to your GitHub repository for evaluation.

Evaluation Criteria:

  1. Functionality (Both Parts): All containers run without errors. The application is accessible via Nginx on http://localhost:5423. The /health and /data routes function correctly. MySQL data persists across container restarts (testable via Compose primarily).
  2. Part 1 Script: The script correctly creates networks/volume, builds images, and runs containers in a logical sequence to achieve the target architecture. Cleanup commands are present.
  3. Part 2 Compose File: The compose.yaml file accurately defines the services, networks, volumes, ports, builds, and dependencies.
  4. Healthchecks & Dependencies (Part 2): Correct implementation of the MySQL Dockerfile healthcheck, the app Compose healthcheck, and depends_on conditions ensuring proper startup order.
  5. Configuration: Dockerfiles are correct. The app Dockerfile correctly installs runtime/dependencies on Alpine. Nginx is properly configured as a reverse proxy. MySQL setup allows the app to connect.
  6. Adherence: Naming conventions, port mappings, network connections, volume usage, directory structure, and the Alpine base image requirement for the app match the specifications.
  7. Repository: Code is submitted via a public GitHub repository with a helpful README.
