Docker-K8s

Hello Kind

This tutorial shows you how to run a sample app on Kubernetes using kind (Kubernetes IN Docker). The tutorial provides a container image that uses NGINX (via agnhost) to echo back all the requests.

Objectives

  • Deploy a sample application to a kind cluster.
  • Run the app.
  • View application logs.
  • Access the application from your local machine.

Prerequisites

This tutorial assumes that you have already set up kind. See the Installation section in the kind Quick Start guide.

You also need to install kubectl. See Install tools for installation instructions. (Note: Adjusted link assuming general K8s docs context, modify if needed).

Create a kind cluster

If you don't already have a kind cluster running, create one now:

kind create cluster

This command creates a default cluster named kind. Your kubectl context will automatically be set to this new cluster (kind-kind). If you want to use a different name, you can specify it with kind create cluster --name <your-cluster-name>.

Create a Deployment

A Kubernetes Pod is a group of one or more Containers, tied together for the purposes of administration and networking. The Pod in this tutorial has only one Container. A Kubernetes Deployment checks on the health of your Pod and restarts the Pod's Container if it terminates. Deployments are the recommended way to manage the creation and scaling of Pods.

  1. Use the kubectl create command to create a Deployment that manages a Pod. The Pod runs a Container based on the provided container image.

    # Run a test container image that includes a webserver
kubectl create deployment hello-node --image=registry.k8s.io/e2e-test-images/agnhost:2.39 -- /agnhost netexec --http-port=8080

  2. View the Deployment:

    kubectl get deployments

    The output is similar to:

    NAME         READY   UP-TO-DATE   AVAILABLE   AGE
hello-node   1/1     1            1           1m

    (It may take some time for the pod to become available. If you see "0/1" under READY, try again in a few seconds.)

  3. View the Pod:

    kubectl get pods

    The output is similar to:

    NAME                          READY   STATUS    RESTARTS   AGE
hello-node-5f76cf6ccf-br9b5   1/1     Running   0          1m

    (The exact pod name will differ)

  4. View cluster events:

    kubectl get events

  5. View the kubectl configuration (optional, shows the current context pointing to your kind cluster):

    kubectl config view

  6. View application logs for the container in the pod.

    Note: Replace hello-node-5f76cf6ccf-br9b5 in the kubectl logs command below with the actual name of the pod from the kubectl get pods command output you saw earlier.

    # Example using the pod name from step 3:
kubectl logs hello-node-5f76cf6ccf-br9b5

    The output is similar to:

    I0911 09:19:26.677397       1 log.go:195] Started HTTP server on port 8080
I0911 09:19:26.677586       1 log.go:195] Started UDP server on port 8081

Note: For more information about kubectl commands, see the kubectl overview.

Create a Service

By default, the Pod is only accessible by its internal IP address within the Kubernetes cluster. To make the hello-node Container accessible from outside the Kubernetes virtual network (specifically, from your local machine when using kind), you have to expose the Pod as a Kubernetes Service.

Warning: The agnhost container has a /shell endpoint, which is useful for debugging, but dangerous to expose to the public internet. Do not run this on an internet-facing cluster, or a production cluster. This tutorial is safe because we will only access it locally via port-forwarding.

  1. Expose the Pod using the kubectl expose command:

    kubectl expose deployment hello-node --type=LoadBalancer --port=8080
    • The --type=LoadBalancer flag indicates that you want to expose your Service. While kind doesn't provision a cloud load balancer, using this type allows tools or integrations (like MetalLB, if installed, though not needed for this tutorial) to assign an external IP or makes it selectable for port-forwarding.
    • The application code inside the test image only listens on TCP port 8080. kubectl expose correctly maps the Service port 8080 to the Pod's targetPort 8080 by default when --port matches the container port.

  2. View the Service you created:

    kubectl get services

    The output is similar to:

    NAME         TYPE           CLUSTER-IP      EXTERNAL-IP   PORT(S)          AGE
hello-node   LoadBalancer   10.96.152.149   <pending>     8080:3xxxx/TCP   21s
kubernetes   ClusterIP      10.96.0.1       <none>        443/TCP          23m

    Unlike cloud providers, kind does not automatically assign an EXTERNAL-IP to LoadBalancer services. The status will remain <pending>. To access the service from your host machine, you need to use kubectl port-forward.

  3. Forward a local port to the service port. Open a new terminal window (as this command blocks) and run:

    # Forward local port 8080 to the service's port 8080
kubectl port-forward service/hello-node 8080:8080

    You should see output like: Forwarding from 127.0.0.1:8080 -> 8080 Forwarding from [::1]:8080 -> 8080

    Leave this terminal running.

  4. Now, access the application. Open your web browser and go to http://localhost:8080, or use curl in another terminal:

    curl http://localhost:8080

    You should see the output from the agnhost application, echoing details about the request (hostname, client IP, etc.).

Clean up

Now you can clean up the Kubernetes resources you created in your cluster:

# Stop the port-forwarding process (Ctrl+C in its terminal)

# Delete the service
kubectl delete service hello-node

# Delete the deployment
kubectl delete deployment hello-node

If you are finished using the kind cluster, you can delete it entirely:

# Delete the default kind cluster
kind delete cluster

If you created a cluster with a specific name (e.g., kind create cluster --name my-cluster), delete it using that name:

# Optional: Delete a specifically named cluster
# kind delete cluster --name my-cluster

Conclusion

This page covered the basic steps to get a kind cluster up and running and deploy a simple application to it. You learned how to create deployments and services, view logs, and access your application locally using port-forwarding. You are now ready to deploy more complex applications to your local kind cluster.
