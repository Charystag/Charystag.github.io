Docker-K8s

TP — Deux conteneurs qui se parlent (sans docker-compose)

Objectif

Faire tourner deux conteneurs Docker dans un même réseau personnalisé, et les faire communiquer directement par leur nom, sans utiliser d’IP.

Contexte

Tu vas créer deux conteneurs :

  1. Un serveur HTTP minimal (ou TCP si tu préfères)
  2. Un client qui va s’y connecter et afficher la réponse

Le tout sans Compose.

Contraintes

  • Pas de docker-compose
  • Les conteneurs doivent être connectés à un réseau custom
  • Le client doit pouvoir parler au serveur via son nom Docker
  • Pas de localhost, pas d’IP en dur, juste le nom du conteneur

À faire

  1. Crée une image server avec une app qui écoute sur un port (ex : Python Flask sur 5000 ou Node/Express)
    • Réponds juste “Hello client” sur /
  2. Crée une image client avec un script qui fait une requête HTTP vers http://server:5000/
    • Utilise curl, wget, httpie, ou un script custom

Bonus

  • Ajoute une variable d’environnement dans le client pour définir le nom du serveur (SERVER_URL)

À avoir

  • Le code du serveur + client
