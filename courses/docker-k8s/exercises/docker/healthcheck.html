Docker-K8s

TP : Ton Conteneur Couteau Suisse (Entrypoint & Healthcheck)

Objectif

On va voir deux trucs stylés :

  1. Comment créer une image Docker qui agit comme un outil en ligne de commande que tu peux appeler direct (ENTRYPOINT). Fini les docker run ubuntu bash -c "ma commande" !
  2. Comment ajouter un check de santé (HEALTHCHECK) à une image pour que Docker sache si ton service à l'intérieur est opérationnel.

Prérequis

  • Docker fonctionnel (obviously).
  • Savoir écrire un Dockerfile simple.

Partie 1 : Le Conteneur-Outil avec ENTRYPOINT

On va faire une image qui fait parler une vache pour ca, on va utiliser cowsay.

  1. Crée un Dockerfile.cowsay :
    • Pars d'une image légère, genre alpine:latest
    • Mets à jour les paquets et installe cowsay. Sur Alpine, c'est un peu différent, il faut souvent chercher le bon paquet ou passer par d'autres dépôts (ou prendre une base Debian/Ubuntu si Alpine est trop relou pour cowsay, mais Alpine c'est bien pour la légèreté).
      • Tu peux utiliser le script install_cowsay.sh present en annexe de l'exercice (attention, tu devras l'adapter si tu utilises alpine)
    • LA LIGNE CLÉ : Définis l'entrypoint pour que le conteneur devienne la commande cowsay : 
      ENTRYPOINT ["cowsay"]
# On peut ajouter un CMD par défaut si on veut (message par défaut si on lance sans args)
# CMD ["Default message if no args"]
  2. Build l'image : docker build -t mon-cowsay -f Dockerfile.cowsay .
  3. Teste comme une commande :
    • Lance direct avec un message : docker run --rm mon-cowsay "Docker parle!" Tu devrais voir la vache parler.
    • Lance avec une option de cowsay (pour changer l'animal, -f) : docker run --rm mon-cowsay -f dragon "Je crache du feu sur tes bugs"

Partie 2 : Le Check de Santé avec HEALTHCHECK

On va créer un serveur web ultra-basique et lui ajouter un check de santé.

  1. Crée un Dockerfile.webhealth :

    • Pars d'une image alpine ou ubuntu
    • Installe curl (qui servira pour le healthcheck) : RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/* (si base Debian/Ubuntu) ou RUN apk add --no-cache curl (si base Alpine).
    • Installe python
    • Ajoute une commande CMD pour lancer un serveur web simple sur le port 8000. Le plus simple avec Python, c'est : 
      # ... après install curl ...
WORKDIR /app # Juste pour avoir un dossier
EXPOSE 8000
CMD ["python3", "-m", "http.server", "8000"]
    • Ajoute le HEALTHCHECK : On va demander à Docker de vérifier toutes les X secondes si le serveur répond bien sur le port 8000. 
      # ... après CMD ...
HEALTHCHECK --interval=10s --timeout=3s --start-period=5s --retries=3 \
  CMD curl --fail http://localhost:8000/ || exit 1
      • --interval=10s : Vérifie toutes les 10 secondes.
      • --timeout=3s : Si la commande curl prend plus de 3s, c'est considéré comme échec.
      • --start-period=5s : Laisse 5 secondes au conteneur pour démarrer avant de commencer les checks (évite les fails au tout début).
      • --retries=3 : Il faut 3 échecs consécutifs pour marquer le conteneur comme unhealthy.
      • CMD curl ... || exit 1 : La commande de check. curl --fail retourne un code d'erreur si le HTTP status n'est pas 2xx. Le || exit 1 assure qu'on renvoie bien un code d'erreur si curl échoue.

  2. Build l'image : docker build -t web-health -f Dockerfile.webhealth .

  3. Observe la santé :

    • Lance le conteneur : docker run -d --name health-test web-health
    • Regarde l'état avec docker ps : 
      docker ps # Regarde la colonne STATUS
      Au début, tu verras peut-être (health: starting). Après quelques secondes (le start-period), ça devrait passer à (healthy).
    • (Bonus) Simule un problème : Connecte-toi au conteneur (docker exec -it health-test sh ou bash) et tue le processus Python (pkill python). Regarde à nouveau docker ps après quelques checks ratés (ça prendra interval * retries secondes), il devrait passer à (unhealthy).
    • Clean : docker rm -f health-test.

apt -y install curl unzip perl
