Introduction to useEffect

In React, the useEffect hook allows you to handle side effects in your components. Side effects include:

  • Fetching data from an API
  • Subscribing to and unsubscribing from events
  • Manipulating the DOM
  • Setting up or cleaning up timers (e.g., setTimeout, setInterval)

Before useEffect, handling side effects in function components was more complex, often requiring class-based components and lifecycle methods like componentDidMount and componentDidUpdate. However, with hooks, we can now perform side effects in function components in a much simpler way.

Basic Syntax

The useEffect hook takes two arguments:

useEffect(effectFunction, dependencyArray);
  1. effectFunction: A function that contains the side effect code (e.g., fetching data, updating the title).
  2. dependencyArray (optional): An array of values that determine when the effect should run.

Running an Effect on Every Render

If you don’t provide a dependency array, the effect runs after every render:

import { useEffect, useState } from "react";

function ExampleComponent() {
    const [count, setCount] = useState(0);

    useEffect(() => {
        console.log("Effect ran after render");
    });

    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>Increment</button>
        </div>
    );
}

export default ExampleComponent;

:warning: Be careful when omitting the dependency array!
If the effect runs on every render, it can lead to performance issues.

Running an Effect Only Once (on Mount)

If you pass an empty dependency array ([]), the effect runs only once, when the component mounts:

useEffect(() => {
    console.log("Component mounted!");
}, []); // Runs only once

This is useful for fetching data from an API when a component loads.

Running an Effect Only When Certain Values Change

If you provide a dependency array with variables, the effect only runs when those values change:

useEffect(() => {
    console.log(`Count changed: ${count}`);
}, [count]); // Runs only when `count` changes

Example:

import { useEffect, useState } from "react";

function ExampleComponent() {
    const [count, setCount] = useState(0);
    const [text, setText] = useState("");

    useEffect(() => {
        console.log(`Count updated: ${count}`);
    }, [count]); // Runs when `count` changes, but NOT when `text` changes

    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>Increment</button>
            <input type="text" onChange={(e) => setText(e.target.value)} />
        </div>
    );
}

export default ExampleComponent;

:bulb: You can pass multiple dependencies:

useEffect(() => {
  console.log("Either count or text changed");
}, [count, text]); // Runs when `count` OR `text` changes

Cleaning Up Effects

Sometimes, effects need cleanup to avoid memory leaks. For example, if you're setting up an event listener or a timer, you need to remove it when the component unmounts.

To clean up an effect, return a function inside useEffect:

useEffect(() => {
    console.log("Effect started");

    return () => {
        console.log("Cleanup before unmounting or updating");
    };
}, []);
