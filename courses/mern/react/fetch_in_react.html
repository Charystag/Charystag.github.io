Fetching Data from a Backend in a React Component (Dockerized Setup)

When running a React frontend and an Express backend in separate Docker containers on the same network, we must consider cross-container communication, CORS, and how to correctly fetch data from the backend.

Backend Setup (express-container on Port 5000)

The backend runs inside a Docker container with the hostname express-container, listening on port 5000.

Setting Up CORS in Express

Since the frontend and backend are running in different containers, we must configure CORS (Cross-Origin Resource Sharing) on the backend to allow requests from the React app.

:warning: By default, browsers block cross-origin requests unless the server explicitly allows them.

To enable CORS in Express, install the cors middleware:

npm install cors

Then, modify your Express server (server.js) to allow requests from the React frontend:

import express from 'express';
import cors from 'cors';

const app = express();

// Allow CORS for requests coming from the frontend container
app.use(
    cors({
        origin: "http://localhost:8080", // The address your browser is connecting to 
        methods: ["GET", "POST", "PUT", "DELETE"],
        allowedHeaders: ["Content-Type", "Authorization"],
    })
);

app.listen(5000, () => console.log("Backend running on port 5000"));

:bulb: Setting origin: "http://localhost:8080" ensures that only the frontend can access the API.
To allow all origins (not recommended in production), use origin: "*".

Docker Network Setup

Both containers should be part of the same Docker network so that React can reach Express using http://express-container:5000.

Fetching Data in a React Component

Now that the backend is reachable at http://express-container:5000, we can fetch data inside a React component.

Creating the ProductList Component

Let's create components/ProductList.tsx and fetch product data from the backend.

import { useEffect, useState } from "react";

interface Product {
    name: string;
    price: number;
    image: string;
}

function ProductList() {
    const [products, setProducts] = useState<Product[]>([]);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        fetch("http://localhost:5000/get/products") // Using backend container name
            .then((response) => {
                if (!response.ok) {
                    throw new Error(`HTTP error! Status: ${response.status}`);
                }
                return response.json();
            })
            .then((data) => {
                setProducts(data);
                setLoading(false);
            })
            .catch((error) => {
                console.error("Error fetching products:", error);
                setLoading(false);
            });
    }, []);

    if (loading) {
        return <p>Loading products...</p>;
    }

    return (
        <div>
            <h2>Product List</h2>
            <ul>
                {products.map((product, index) => (
                    <li key={index}>
                        <h3>{product.name}</h3>
                        <p>Price: ${product.price}</p>
                        <img src={product.image} alt={product.name} width={100} />
                    </li>
                ))}
            </ul>
        </div>
    );
}

export default ProductList;

:warning: Why use http://express-container:5000 instead of localhost?
When running inside Docker, localhost refers to the container itself, not the backend. Instead, we use express-container, which is the hostname of the backend inside the Docker network.

Using ProductList in the App Component

Now, we can import and use the ProductList component inside App.tsx.

import ProductList from "./components/ProductList.tsx";

function App() {
    return (
        <div>
            <h1>Welcome to the Store</h1>
            <ProductList />
        </div>
    );
}

export default App;

Handling CORS Issues Inside Docker

If your frontend cannot reach the backend due to CORS issues, ensure:

  1. The backend allows requests from the correct origin (http://frontend-container:5173).
  2. The frontend is correctly fetching from http://express-container:5000, not localhost.
  3. The backend container is reachable from the frontend container by checking:
docker exec -it react-container ping express-container

If the backend is unreachable, check your Docker network settings.

Alternative: Using Environment Variables for API URLs

Instead of hardcoding http://express-container:5000, you can store the API URL in an environment variable.

Create a .env File in the Frontend

BACKEND_URL=http://express-container:5000

Access It in ProductList.tsx

const API_URL = process.env.BACKEND_URL;

useEffect(() => {
    fetch(`${API_URL}/get/products`)

:bulb: Why use environment variables?
This makes it easier to switch between environments (e.g., development, production) without changing the code.

Now, your React frontend can successfully fetch data from the backend running in a separate Docker container. If you experience any issues, check:
