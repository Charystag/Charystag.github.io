Introduction to Fetch in JavaScript

When working with APIs in JavaScript, the fetch API provides a simple and modern way to make network requests. It allows us to retrieve resources asynchronously, typically in the form of JSON data from a server.

The fetch function returns a Promise, meaning it works asynchronously and resolves with a Response object.

Basic Usage

Here’s a simple example of fetching data from a public API:

fetch("https://jsonplaceholder.typicode.com/todos/1")
    .then((response) => response.json()) // Convert response to JSON
    .then((data) => console.log(data)) // Handle the data
    .catch((error) => console.error("Error:", error)); // Handle errors

In the example above:

  • We make a GET request to the API.
  • We convert the response to JSON using .json(), which also returns a Promise.
  • We log the result to the console.
  • We handle any potential errors with .catch().

Using async/await

Using async/await makes the code easier to read and avoids deep nesting:

async function fetchData() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/todos/1");
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Error:", error);
    }
}

fetchData();

:bulb: Unlike .then(), using async/await allows us to write asynchronous code in a synchronous style.

Handling Errors

The fetch API only rejects a Promise if there is a network failure (e.g., no internet connection). If the request is successful but returns an HTTP error (like 404 or 500), fetch does not reject the promise. To handle such cases, we need to check response.ok:

async function fetchData() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/todos/1");

        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Fetch error:", error);
    }
}

fetchData();

:warning: Always check response.ok before processing the response.

Making Different Types of Requests (GET, POST, PUT, DELETE)

By default, fetch makes a GET request, but you can specify different HTTP methods like POST, PUT, and DELETE using the RequestInit object.

Sending a POST Request with fetch

A POST request is used to send data to a server. This typically requires a request body in JSON format and specifying the appropriate headers.

async function createPost() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/posts", {
            method: "POST",
            headers: {
                "Content-Type": "application/json",
            },
            body: JSON.stringify({
                title: "New Post",
                body: "This is a new post",
                userId: 1,
            }),
        });

        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        const data = await response.json();
        console.log("Created post:", data);
    } catch (error) {
        console.error("Fetch error:", error);
    }
}

createPost();

:bulb: Always set the Content-Type header to "application/json" when sending JSON data in a POST request.

Sending a PUT Request

A PUT request is used to update an existing resource.

async function updatePost(postId) {
    try {
        const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${postId}`, {
            method: "PUT",
            headers: {
                "Content-Type": "application/json",
            },
            body: JSON.stringify({
                id: postId,
                title: "Updated Post",
                body: "This post has been updated",
                userId: 1,
            }),
        });

        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        const data = await response.json();
        console.log("Updated post:", data);
    } catch (error) {
        console.error("Fetch error:", error);
    }
}

updatePost(1);

:warning: PUT typically replaces the entire resource, so all fields must be included in the request body.

Sending a DELETE Request

A DELETE request removes a resource from the server.

async function deletePost(postId) {
    try {
        const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${postId}`, {
            method: "DELETE",
        });

        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        console.log(`Post ${postId} deleted successfully`);
    } catch (error) {
        console.error("Fetch error:", error);
    }
}

deletePost(1);

:bulb: Unlike POST or PUT, a DELETE request typically does not require a request body.

Handling Timeouts in fetch

The fetch API does not have a built-in timeout mechanism. If a request hangs indefinitely, you need to use the AbortController API.

async function fetchWithTimeout(url, timeout = 5000) {
    const controller = new AbortController();
    const signal = controller.signal;

    setTimeout(() => controller.abort(), timeout);

    try {
        const response = await fetch(url, { signal });

        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        const data = await response.json();
        console.log(data);
    } catch (error) {
        if (error.name === "AbortError") {
            console.error("Request timed out");
        } else {
            console.error("Fetch error:", error);
        }
    }
}

fetchWithTimeout("https://jsonplaceholder.typicode.com/todos/1", 3000);

:warning: If the request takes longer than timeout, the request will be aborted.

Setting Custom Headers

Sometimes, you need to send additional headers in your request, such as an Authorization token.

async function fetchWithHeaders() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/todos/1", {
            headers: {
                "Authorization": "Bearer YOUR_ACCESS_TOKEN",
                "Accept": "application/json",
            },
        });

        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Fetch error:", error);
    }
}

fetchWithHeaders();

:bulb: Custom headers are required for authentication and setting request preferences.

Handling CORS Issues

When fetching data from a different origin, you may encounter CORS (Cross-Origin Resource Sharing) errors. These errors occur because the browser enforces same-origin policy.

  • If you're fetching from an Express server, you need to enable CORS on the backend:
const cors = require("cors");
const express = require("express");

const app = express();
app.use(cors());

app.get("/get/products", (req, res) => {
    res.json([{ name: "Laptop", price: 1200, image: "laptop.jpg" }]);
});

app.listen(5000, () => console.log("Server running on port 5000"));

:warning: Never use cors() with Access-Control-Allow-Origin: '*' in production unless it's truly necessary.

Sending URL Parameters with fetch

If an API requires query parameters, use URLSearchParams:

async function fetchWithParams() {
    const params = new URLSearchParams({ userId: 1 });

    const response = await fetch(`https://jsonplaceholder.typicode.com/todos?${params.toString()}`);
    const data = await response.json();
    console.log(data);
}

fetchWithParams();

:bulb: This method is useful when filtering or paginating API results.

Downloading and Processing Binary Data (Images, PDFs)

To download and process binary data (e.g., images, PDFs), use Response.blob():

async function fetchImage() {
    const response = await fetch("https://via.placeholder.com/150");
    const blob = await response.blob();
    const imgUrl = URL.createObjectURL(blob);

    const img = document.createElement("img");
    img.src = imgUrl;
    document.body.appendChild(img);
}

fetchImage();

:bulb: blob() allows handling files, such as downloading PDFs or images.
