Visualizing Prompts

Parameter Definitions

  • Audience: Describes the skill level and profile of the target learner.

    • Beginner: Simple, direct, no assumptions. Introduce the most basic concepts with clarity.
    • Intermediate: Build on fundamentals. The learner can interpret and apply basic concepts, handle simple problem-solving, and understand common terminology.
    • Advanced: Complex, multi-concept, potentially tricky scenarios requiring deep understanding and analysis. The learner can handle subtlety, debugging, and less common features.

  • Background: Provides context about the learner’s environment or prior knowledge.

    • Example: "No prior programming experience" means you must avoid assuming they know any programming terms. "Familiar with Python basics" allows using Python syntax and basic constructs.

  • Programming Language: Specifies the main language or technology context for the question.

    • Example: "Python", "Java", "AWS", "Azure", or null if the question should be language-agnostic.
    • If a programming language or platform is specified, the question should use relevant syntax, terminology, or services. If null, the question remains conceptually focused without language-specific syntax.

  • Topic: The subject matter or concept the question focuses on. This can be a language feature (e.g., Loops, Data Structures), a platform-related subject (e.g., S3, EC2, Virtual Networks), or more general programming concepts.

  • If a specific topic is provided, the question should directly relate to it.

  • If null, then the question should remain broadly relevant to the chosen programming language or context without focusing on a particular subtopic.

  • Difficulty: Indicates the complexity level of the question. The question must match the specified difficulty.

    • Easy: Simple recall or recognition tasks suitable for complete beginners. Typically involves direct, obvious outcomes (e.g., output of print(2+3)).
    • Medium: Requires applying known concepts in a straightforward scenario.
      • Straightforward means no hidden tricks, obscure language features, or unexpected behavior.
      • Example: If the learner knows loops and arithmetic, a medium question might ask: "What does the following loop print?", where the loop and arithmetic are directly understandable without guessing unusual behavior.
      • Non-straightforward (which should NOT be in medium) might involve subtle language quirks, odd data structures rarely encountered, or performance optimizations not expected at this level.
    • Hard: Demands advanced problem-solving. This might involve multiple concepts together, require understanding edge cases, debugging code, or interpreting less common language features. For example, analyzing nested loops with conditional logic that produce a non-obvious result, or identifying why a code snippet raises a TypeError due to a subtle misuse of types.

  • Type: Specifies the format of the question.

    • Multiple Choice: The final output is a question expecting multiple-choice answers. Do not provide the answers, only the question prompt.
    • True/False: The final output is a statement where the student decides if it’s true or false. Only output the statement.

  • Purpose: Defines the intent of the question.

    • Conceptual: Tests definitions, theory, or syntax without requiring execution or problem-solving. (e.g., "Which of these keywords defines a function in Python?")
    • Problem Solving: Requires applying concepts to derive an answer. (e.g., "What will be printed by this loop?")
    • Implementation: Focuses on real-world coding tasks, debugging, or understanding practical use cases. (e.g., "Which code snippet correctly initializes a Flask route?")
