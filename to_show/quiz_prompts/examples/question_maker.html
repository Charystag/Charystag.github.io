Example 1: Beginner, Conceptual, Python Data Types

Input:

{
  "rules": {
    "audience": "beginner",
    "background": "No prior programming experience.",
    "programming_language": "Python",
    "topic": "Data Types",
    "difficulty": "easy",
    "type": "multiple_choice",
    "purpose": "conceptual"
  },
  "previous_questions": [
    "What is the output of print(2 + 2)?",
    "Which of these is a valid Python variable name?"
  ]
}

Rationale:

  • Audience & Background: Beginner with no experience → stick to very basic concepts.
  • Programming Language: Python → use Pythonic terminology.
  • Topic: Data Types → introduce Python data types, a fundamental concept.
  • Difficulty: Easy → simple recall or recognition, no tricky aspects.
  • Purpose: Conceptual → testing understanding of what data types exist in Python.
  • Novelty: Previously tested arithmetic and variable naming; now test recognition of a specific data type form.

Output:

{
  "question": "Which of the following represents a string data type in Python?",
  "type": "multiple_choice"
}

This question shifts focus from arithmetic and variable names to identifying a data type (a string), encouraging learning and variety.

Example 2: Intermediate, Problem Solving, Python Loops

Input:

{
  "rules": {
    "audience": "intermediate",
    "background": "Familiar with Python basics.",
    "programming_language": "Python",
    "topic": "Loops",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "problem_solving"
  },
  "previous_questions": [
    "What does the following code print? for i in range(3): print(i * 2)"
  ]
}

Rationale:

  • Audience & Background: Intermediate with Python basics → can handle loops and simple conditions.
  • Topic: Loops, and the previous question already tested a basic loop’s arithmetic output.
  • Difficulty: Medium → apply known concepts straightforwardly, no hidden complexity.
  • Purpose: Problem Solving → determine output by reading and understanding code.
  • Novelty: Instead of just arithmetic in a loop, introduce a condition to encourage a slightly different scenario.

Output:

{
  "question": "What will this code print?\n\nnumbers = [2, 4, 6, 8]\nfor n in numbers:\n    if n % 4 == 0:\n        print(n)",
  "type": "multiple_choice"
}

This code tests the learner’s ability to understand loops with a simple condition (modulus check). It’s still straightforward but more involved than a pure arithmetic loop.

Example 3: Advanced, Implementation, AWS Load Balancing

Input:

{
  "rules": {
    "audience": "advanced",
    "background": "Experienced with AWS services.",
    "programming_language": "AWS",
    "topic": "Load Balancing",
    "difficulty": "hard",
    "type": "multiple_choice",
    "purpose": "implementation"
  },
  "previous_questions": [
    "Which AWS service stores files as objects in a bucket?",
    "What does print(sum([1, 2, 3])) output in Python?"
  ]
}

Rationale:

  • Audience & Background: Advanced and AWS-experienced → can assume familiarity with AWS services.
  • Topic: Load Balancing → a more complex AWS concept than basic storage.
  • Difficulty: Hard → involves understanding a real-world scenario: routing requests at scale.
  • Purpose: Implementation → practical usage of AWS load balancing solutions.
  • Novelty: Previously asked about basic AWS storage (S3) and a simple Python sum. Now we introduce load balancing, requiring knowledge of AWS’s application-level routing.

Output:

{
  "question": "You have a microservices application running on multiple EC2 instances. Which AWS service would you use to distribute incoming HTTP requests based on URL paths?",
  "type": "multiple_choice"
}

This question targets Application Load Balancing—a more advanced and scenario-based concept, encouraging practical understanding.

Example 4: Intermediate, Conceptual, Java Syntax

Input:

{
  "rules": {
    "audience": "intermediate",
    "background": "Understands basic Java programming.",
    "programming_language": "Java",
    "topic": "Syntax",
    "difficulty": "medium",
    "type": "true_false",
    "purpose": "conceptual"
  },
  "previous_questions": [
    "Which keyword defines a function in Python?",
    "True or False: print(2+2) in Python returns 4?"
  ]
}

Rationale:

  • Audience & Background: Intermediate and familiar with Java basics.
  • Topic: Java Syntax → test a well-known rule in Java.
  • Difficulty: Medium → straightforward conceptual understanding.
  • Purpose: Conceptual → no code execution needed, just a language rule check.
  • Novelty: Switches language from Python to Java and tests a different concept entirely (Java’s class-file naming rule), ensuring new learning.

Output:

{
  "question": "True or False: In Java, a public class must be defined in a file sharing its exact name.",
  "type": "true_false"
}

This question tests a key Java convention, distinct from previous Python-based questions.
