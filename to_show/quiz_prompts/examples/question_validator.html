Example 1: Valid Question Aligned with Parameters

Scenario:

  • Audience: beginner, background: "No prior programming experience."
  • Programming_language: "Python", Topic: "Data Types"
  • Difficulty: easy, Type: multiple_choice, Purpose: conceptual
  • Previous questions: tested arithmetic output and variable naming only.

Input:

{
  "rules": {
    "audience": "beginner",
    "background": "No prior programming experience.",
    "programming_language": "Python",
    "topic": "Data Types",
    "difficulty": "easy",
    "type": "multiple_choice",
    "purpose": "conceptual"
  },
  "question": "Which of the following represents a string in Python?",
  "previous_questions": [
    "What is the output of print(2 + 2)?",
    "Which of these is a valid Python variable name?"
  ]
}

Expected Output:

{
  "validation": true,
  "feedback": null
}

Rationale:

  • The question is easy and conceptual (just identifying a string).
  • It’s relevant to Python data types and not previously asked.
  • The format is multiple_choice and no answers are provided.

Example 2: Question Not Relevant to Topic

Scenario:

  • Audience: intermediate, background: "Familiar with Python basics."
  • Programming_language: "Python", Topic: "Loops"
  • Difficulty: medium, Type: multiple_choice, Purpose: problem_solving
  • Previous question tested a simple arithmetic loop.

Input:

{
  "rules": {
    "audience": "intermediate",
    "background": "Familiar with Python basics.",
    "programming_language": "Python",
    "topic": "Loops",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "problem_solving"
  },
  "question": "Which of the following data types is immutable in Python?",
  "previous_questions": [
    "What does the following code print?\nfor i in range(3): print(i * 2)"
  ]
}

Expected Output:

{
  "validation": false,
  "feedback": "The question is not aligned with the specified topic 'Loops'. It should involve iteration or looping constructs in Python."
}

Rationale:

  • The parameters require a problem-solving question about loops.
  • The provided question tests knowledge of immutable data types instead of loops, failing the relevance requirement.

Example 3: Question Too Trivial for the Difficulty

Scenario:

  • Audience: intermediate, background: "Has written simple Python functions."
  • Programming_language: "Python", Topic: "Functions"
  • Difficulty: medium, Type: multiple_choice, Purpose: problem_solving
  • The question should require applying a known concept straightforwardly, not just recalling a fact.

Input:

{
  "rules": {
    "audience": "intermediate",
    "background": "Has written simple Python functions.",
    "programming_language": "Python",
    "topic": "Functions",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "problem_solving"
  },
  "question": "Which keyword is used to define a function in Python?",
  "previous_questions": [
    "True or False: print(2+2) returns 4?"
  ]
}

Expected Output:

{
  "validation": false,
  "feedback": "The question is too simple for a 'medium' difficulty problem_solving scenario. It only asks for a keyword recall instead of requiring the learner to apply or interpret functions in context."
}

Rationale:

  • At medium difficulty and with a problem_solving purpose, the question should not just ask which keyword defines a function.
  • It should require interpreting or applying a function concept, e.g., understanding what a function returns or analyzing a function’s behavior.

Example 4: Question Gives Away an Answer or Explanation

Scenario:

  • Audience: advanced, background: "Experience with AWS"
  • Programming_language: "AWS", Topic: "S3"
  • Difficulty: hard, Type: multiple_choice, Purpose: implementation
  • The question should test implementation-related S3 concepts.

Input:

{
  "rules": {
    "audience": "advanced",
    "background": "Experience with AWS",
    "programming_language": "AWS",
    "topic": "S3",
    "difficulty": "hard",
    "type": "multiple_choice",
    "purpose": "implementation"
  },
  "question": "Which of the following AWS services can store objects in buckets? (Answer: S3)",
  "previous_questions": [
    "Which AWS service is used to run containers at scale?"
  ]
}

Expected Output:

{
  "validation": false,
  "feedback": "The question includes an answer hint '(Answer: S3)' which violates the rule of providing no answers or explanations."
}

Rationale:

  • The question includes an answer hint directly in the prompt, breaking the no-answer rule.

Example 5: Question Too Similar to a Previous One

Scenario:

  • The new question must introduce a novel element not previously tested.

Input:

{
  "rules": {
    "audience": "beginner",
    "background": "No prior programming experience.",
    "programming_language": "Python",
    "topic": "Data Types",
    "difficulty": "easy",
    "type": "multiple_choice",
    "purpose": "conceptual"
  },
  "question": "Which symbol indicates a comment in Python?",
  "previous_questions": [
    "What symbol is used for comments in Python?",
    "Which of these is a valid Python variable name?"
  ]
}

Expected Output:

{
  "validation": false,
  "feedback": "The new question closely resembles a previous one testing the same concept (comments in Python) without introducing a new element."
}

Rationale:

  • The previous questions already tested the exact same concept (comment symbol). This new question is essentially a rephrased duplicate.

Example 6: Null programming_language but a Specific topic

Scenario:

  • programming_language = null
  • topic = "Data Structures"
  • The question should remain language-agnostic and still be relevant to Data Structures.
  • Parameters suggest an intermediate difficulty conceptual question. The question should involve data structures without referencing language-specific syntax.

Input:

{
  "rules": {
    "audience": "intermediate",
    "background": "Understands basic programming concepts.",
    "programming_language": null,
    "topic": "Data Structures",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "conceptual"
  },
  "question": "Which data structure uses a Last-In-First-Out (LIFO) principle?",
  "previous_questions": []
}

Expected Output:

{
  "validation": true,
  "feedback": null
}

Rationale:

  • The question is relevant to the topic (Data Structures) and remains language-agnostic as programming_language is null.
  • For medium difficulty conceptual, identifying a LIFO data structure (like a stack) is straightforward yet not trivial.
  • No issues with novelty or format.

Example 7: Hard Difficulty Question Not Complex Enough

Scenario:

  • programming_language = "Python", topic = "Error Handling"
  • difficulty = "hard", type = "multiple_choice", purpose = "implementation"
  • The question should test advanced problem-solving or less common features. A trivial recall question fails this requirement.

Input:

{
  "rules": {
    "audience": "advanced",
    "background": "Comfortable with Python exceptions and debugging.",
    "programming_language": "Python",
    "topic": "Error Handling",
    "difficulty": "hard",
    "type": "multiple_choice",
    "purpose": "implementation"
  },
  "question": "Which keyword is used to handle exceptions in Python?",
  "previous_questions": []
}

Expected Output:

{
  "validation": false,
  "feedback": "This question is too simple for a hard difficulty implementation scenario. It only tests basic keyword recall rather than demanding advanced problem-solving or nuanced error handling logic."
}

Rationale:

  • Hard difficulty and implementation purpose imply a more complex scenario (e.g., handling specific exceptions or debugging tricky code).
  • Asking which keyword handles exceptions is too trivial.

Example 8: Multiple Violations in One Question

Scenario:

  • Parameters suggest a medium difficulty problem-solving question about loops in Python.
  • The question is irrelevant to loops AND provides an answer hint AND is too trivial.

Input:

{
  "rules": {
    "audience": "intermediate",
    "background": "Familiar with Python basics.",
    "programming_language": "Python",
    "topic": "Loops",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "problem_solving"
  },
  "question": "Which symbol is used for comments in Python? (Answer: #)",
  "previous_questions": [
    "What does this loop print?\nfor i in range(3): print(i * 2)"
  ]
}

Expected Output:

{
  "validation": false,
  "feedback": "Multiple issues detected: The question is not relevant to the 'Loops' topic, it includes an answer hint '(Answer: #)', and it does not meet the medium difficulty or problem_solving requirements."
}

Rationale:

  • The topic is loops, but the question is about comment syntax.
  • It provides a direct answer hint, violating the no-answers rule.
  • It’s too simple and not a problem-solving scenario.
  • The feedback mentions all critical violations to make corrections easier.
