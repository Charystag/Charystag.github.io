Visualizing Prompts

Below are several examples demonstrating how the answer_validator should behave given various inputs. Each example shows the validator’s input (the question, rules, and generated answers) and the expected output (validation result and feedback).

Note: These examples assume that the validator already has access to the global rules and definitions the question_maker and answer_maker followed. The validator’s job is strictly to verify the compliance of the provided answers.

Example 1: Valid Multiple Choice Question

Input:

{
  "question": "Which symbol is used for comments in Python?",
  "rules": {
    "topic": "Python",
    "programming_language": "Python",
    "difficulty": "easy",
    "type": "multiple_choice",
    "purpose": "conceptual",
    "background": "No prior programming experience."
  },
  "answers": {
    "options": [
      {"text": "#", "correct": true},
      {"text": "//", "correct": false},
      {"text": "/*", "correct": false},
      {"text": "--", "correct": false}
    ],
    "explanation": "In Python, the '#' symbol is used to start a comment.",
    "documentation_links": [
      {
        "url": "https://docs.python.org/3/tutorial/introduction.html#comments",
        "description": "Python Official Documentation: Comments"
      }
    ]
  }
}

Expected Output:

{
  "validation": true,
  "feedback": null
}

Rationale:

  • Exactly one correct answer.
  • The explanation aligns with the correct answer.
  • The documentation link is relevant.

Example 2: Multiple Correct Answers in Multiple Choice

Input:

{
  "question": "Which data structure ensures FIFO ordering?",
  "rules": {
    "topic": "Data Structures",
    "programming_language": null,
    "difficulty": "easy",
    "type": "multiple_choice",
    "purpose": "conceptual",
    "background": "Understands basic programming concepts."
  },
  "answers": {
    "options": [
      {"text": "A queue", "correct": true},
      {"text": "A stack", "correct": true},
      {"text": "A linked list", "correct": false},
      {"text": "A binary tree", "correct": false}
    ],
    "explanation": "A queue ensures FIFO ordering, while a stack is LIFO.",
    "documentation_links": []
  }
}

Expected Output:

{
  "validation": false,
  "feedback": "Multiple options are marked as correct in a multiple-choice question. Only one correct answer is allowed."
}

Rationale:

  • Two correct answers (A queue and A stack) violates the single-correct-answer rule for multiple choice.

      {"text": "False", "correct": false}
    ],
    "explanation": "Lists in Python are defined using curly braces {}.",
    "documentation_links": []
  }

{
  "validation": false,
  "feedback": "The explanation does not justify the correct answer. It claims lists use curly braces, but the correct answer is True for square brackets."

  • The correct answer is True (square brackets), but the explanation contradicts this, making it invalid.

Example 4: Irrelevant Documentation Link

Input:

{
  "question": "Which AWS service is used to store objects in buckets?",
  "rules": {
    "topic": "S3",
    "programming_language": "AWS",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "conceptual",
    "background": "Knows basic AWS services."
  },
  "answers": {
    "options": [
      {"text": "Amazon S3", "correct": true},
      {"text": "Amazon EC2", "correct": false},
      {"text": "Amazon RDS", "correct": false},
      {"text": "AWS Lambda", "correct": false}
    ],
    "explanation": "Amazon S3 stores data as objects in buckets.",
    "documentation_links": [
      {
        "url": "https://docs.aws.amazon.com/ec2/",
        "description": "AWS EC2 Documentation"
      }

Example 6: Multiple Issues (Wrong Answer and Poor Explanation)

Input:

{
  "question": "Which of these code snippets prints 'Hello, World' in Python?",
  "rules": {
    "topic": "Python",
    "programming_language": "Python",
    "difficulty": "easy",
    "type": "multiple_choice",
    "purpose": "implementation",
    "background": "No prior programming experience."
  },
  "answers": {
    "options": [
      {"text": "print('Goodbye, World')", "correct": true},
      {"text": "echo 'Hello, World'", "correct": false},
      {"text": "console.log('Hello, World')", "correct": false},
      {"text": "print('Hello, World')", "correct": false}
    ],
    "explanation": "This code prints 'Goodbye, World' which is close enough.",
    "documentation_links": []
  }
}

Expected Output:

{
  "validation": false,
  "feedback": "The correct answer is incorrect (it prints 'Goodbye, World' not 'Hello, World'). The explanation does not justify the correct answer and is inconsistent with the question."
}

Rationale:

  • The chosen correct answer doesn’t match the question’s requirement.
  • The explanation acknowledges it’s not printing ‘Hello, World’ but claims it’s “close enough,” which is not acceptable.

Example 7: Correct Answer Marked but Actually Incorrect

Input:

{
  "question": "Which of the following code snippets will print the numbers 0 to 2 in Python?",
  "rules": {
    "topic": "Loops",
    "programming_language": "Python",
    "difficulty": "medium",
    "type": "multiple_choice",
    "purpose": "problem_solving",
    "background": "Familiar with Python basics."
  },
  "answers": {
    "options": [
      {
        "text": "for i in range(3):\n    print(i)",
        "correct": false
      },
      {
        "text": "for i in [0, 1, 2]:\n    print(i)",
        "correct": false
      },
      {
        "text": "for i in range(5, 8):\n    print(i)",
        "correct": true
      },
      {
        "text": "while i < 3:\n    print(i)",
        "correct": false
      }
    ],
    "explanation": "The chosen snippet (range(5, 8)) prints 5, 6, 7 which is basically three numbers, so it’s fine.",
    "documentation_links": []
  }
}

Expected Output:

{
  "validation": false,
  "feedback": "The selected correct answer does not print 0 to 2; it prints 5, 6, 7. The explanation fails to justify why this would be acceptable."
}

Rationale:

  • The correct answer option is incorrect. It prints 5, 6, 7 instead of 0, 1, 2.
  • The explanation tries to justify it as “three numbers,” but that does not match the question’s requirement.
